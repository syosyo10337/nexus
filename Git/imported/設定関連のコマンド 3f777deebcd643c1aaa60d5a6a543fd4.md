# バージョンを確認する。

```Bash
$ git version
$ git -v

# ログイン
$ git config –global user.name “syosyo10337”
$ git config –global user.email syosyo.ds@gmail.com

# Gitでパスワードを一定時間保持
# タイムアウトまえの時間も設定
$ git config –global credential.helper “cache –timeout=86400”
```

# コマンドを調べる/マニュアル

```Bash
# マニュアルを表示
$ man git
$ man git-<サブコマンド名>

e.g.)#git logコマンドのマニュアルを表示
$ man git-log


# ギットのヘルプページを表示
$ git help
$ git help <サブコマンド>
```

# リモートの周りの設定

```Bash
# リモートリポジトリのURLを表示(確認)
$ git remote -v

# リモートリポジトリの変更
$ git remote set-url origin <new url>
```

# リポジトリの初期化

初期化自体は`git init`コマンドでやることになりそうですが、

```Bash
# 1からcurrent directoryにリポジトリを作成する
# .gitディレクトリをcurrentディレクトリ下に配置する
$ git init


# GitHub上のリポジトリやローカルのリポジトリを
# current directoryに複製
$ git clone <リモートのURL/リポジトリ>

e.g.)# 自分のPFアプリのクローン
$ git clone https://github.com/syosyo10337/recorda-me.git
```

### リモートリポジトリの初期設定

一般にはローカルに作ったリポジトリを保存するために、リモートリポを別途用意し、`remote add`することが多い。

```Bash
# URLの形式で引数にとったリモートリポを"origin"というailiasをつけて、
# 現在のリポジトリのリモートリポジトリとして設定
$ git remote add origin https://github.com/syosyo10337/<repo_name>
```
