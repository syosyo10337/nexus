# 用語    
- index(/staging area)  
    ファイルの変更のsnapshot(ある時間における状態)を保存(store)して置く場所のこと。
    
- commit  
    indexに追加した変更を、メッセージをつけてリポジトリに記録すること。
    
- (tips)コミットメッセージの書き方のコツ
    
- conflict  
    あるファイルについての変更の差分が競合する際に、Gitがどちらの履歴を選択すべきが選べずに、自動的にmergeできない状態のこと。
    
- コンフリクトが起きたら
        
    1. git diffで差分を確認し、
        
    2. 手動で該当ファイルを編集する。
        
    3. git add /git commitでコミットすることで  
        mergeコミットが作成される。
