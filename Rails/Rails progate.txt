コントローラの全アクションで共通する処理がある場合には、before_action使う。before_actionを用いることで、アクションが呼び出される際に必ずbefore_actionの処理が実行されます。ex)before_action :定義したメソッド　以下にメソッドを定義したり

class ApplicationController < ActionController::Base
	# before_actionにset_current_userメソッドを指定してください

		@current_user = User.find_by(id: session[:user_id])   
	end
	
end

***bcryptを導入した時のログイン処理
フォームに入力されたメールアドレス(ユーザID)に一致するユーザを取得し、入力されたパスワードをハッシュ化(authenticateメソッドを用いて)した値と、そのデータベース上で該当したユーザのパスワード_digestの値が一致するかを調べる。

has_secure_passwordメソッドを有効にすると、authenticateメソッドを使えるようになり,
authenticateメソッドは渡された引数をハッシュ化し、メソッドをつかったオブジェクトのもつpassword_digestの値と一致するかどうかを判定してくれます。


def login
	@user = User.find_by(email: params[:email])
	if @user && @user.authenticate(params[:password])
		session[:user_id] = @user.id
		flash[:notice] = "ログインしました"
		redirect_to("/posts/index")
	else
		@error_message = "メールアドレスまたはパスワードが間違っています"
		@email = params[:email]
		@password = params[:password]
		render("users/login_form")
	end
end

	attr_accessor :first_name, :last_name. :email, #アクセサーによって、@first_nameと@last_name,@emailに対して、getter/setterメソッドが使えるようになった。
