# DB周りのメソッド

```PHP
$result = DB::connection("mysql_slave")->select($query);
```

selectメソッドは常に結果の配列を返します。配列内の各結果は、データベースのレコードを表すPHPのstdClassオブジェクトになります。

Laravelのデータベースクエリビルダは、データベースクエリを作成、実行するための便利で流暢(fluent)なインターフェイスを提供します。ほとんどのデータベース操作をアプリケーションで実行するために使用でき、Laravelがサポートするすべてのデータベースシステムで完全に機能します。

Laravelクエリビルダは、PDOパラメーターバインディングを使用して、SQLインジェクション攻撃からアプリケーションを保護します。クエリバインディングとしてクエリビルダに渡たす文字列をクリーンアップやサニタイズする必要はありません。

## `Illuminate\Support\Facades\DB::table(’<table_name>’)`

テーブル名を指定して、そのテーブルへの**クエリビルダインスタンス**を返す(sqlクエリ生成のためのインスタンス),最終的な結果はget()メソッドなどを通じて取得する

## `→get()`

  
クエリの結果を含む`Illuminate\Support\Collection`インスタンスを返す。それぞれの結果はstdClassのインスタンスで、`stdClass`オブジェクトのプロパティにアクセスすることで、カラムの値にアクセスできます。

```PHP
use Illuminate\Support\Facades\DB;

$users = DB::table('users')->get();

foreach ($users as $user) {
    echo $user->name;
}
```

💡

但し、collectionを扱うには、Laravelの用意するメソッド群を使用する方がいいことが多いですよ。それぐらい強力

### クエリビルダインスタンスのメソッドたち?

### `→where()`

WHERE句にあたるものを、文字列区切りで渡す。

このメソッドの戻り値はクエリビルダインスタンス

```PHP
e.g.)
// nameカラムがJohn
$user = DB::table('users')->where('name', 'John')->first();
```

### `→first()`

を使用することで、単一レコードを取得できる.戻り値としてはstdClassのオブジェクト

それぞれのカラム名に対応するプロパティを持つ

該当がないとき　nullを返す

### `→value()`

行全体が必要なく、単一のカラムの値が欲しい場合には、`value`メソッドを使用してレコードから単一の値を抽出できます。このメソッドは、カラムの値を直接返します。Stringとか

```SQL
$email = DB::table('users')->where('name', 'John')->value('email');
```

### `→find()`

単一行をidをもとに取得する。(`int`)

stdClassのオブジェクトが返されるよね。

- find()は、一致するidが見つからなかった場合は、nullを返します。

- findOrFail()は、一致するidが見つからなかった場合は、エラーを返します。

## カラム値のリストを取得

### `→pluck()`

単一のカラムの値を含む`Illuminate\Support\Collection`インスタンスを取得する場合は、`pluck`  
メソッドを使用します。この例では、ユーザーのタイトルのコレクションを取得します。

引数に、dbのカラム名を指定することで、指定したカラム名の値をもつ配列をプロパティとして持つCollectionインスタンスを取得できる。

それぞれの値はStringなどとして扱われる。

`pluck`メソッドに２番目の引数を指定し、結果のコレクションがキーとして使用する列を指定できます。

```PHP
$titles = DB::table('users')->pluck('title', 'name');

foreach ($titles as $name => $title) {
    echo $title;
}
```

### `→query()`

モデルに対して、このメソッドを使用すると、このモデルに対してのクエリを作成できるクエリビルダインスタンスを返します。あとはwhereでもなんでも
