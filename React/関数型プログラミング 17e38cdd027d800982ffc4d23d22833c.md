# 関数型プログラミングの理解(Functional Programming)

宣言型プログラミングのうちの一つ.

関数を最大限利用することでコードを整理するプログラム手法のこと。オブジェクト指向に対比される。

- 関数型プログラミングとは、なるべく手続き型の制御を関数に分離(隠蔽)し、やりたいことに集中できるようにするプログラミング手法。

➡️

## 手続き型(命令)プログラミング

ブラウザなどへ命令を手順通りに記述していく手法。

- 関数型プログラミングとは、なるべく手続き型の制御を関数に分離(隠蔽)し、やりたいことに集中できるようにするプログラミング手法。

すごく簡易的な例

```TypeScript
e.g.)
let nums = [1,2,3];
let doubleNums = nums.map(num => num * 2);
```

ループ制御:->mapメソッドに担当させる。  
やりたいこと: ->関数で定義。(開発者が担当)

## 関数型プログラミングのメリット(目指すところ)

1. コードの可読性の向上

2. 拡張性/再利用性の向上

3. テスト性の向上

4. モジュール化の向上

5. Tree Shakingの向上(本番用に無駄なコードを削除する機能)

## 関数型プログラミングにおけるキーワード

- (値の)状態管理と処理を分離する。
    
    - e.g. useStateで状態を管理して、コンポーネントで処理を行う
    

- 純粋関数（副作用を排除する）特定の入力には特定の出力を返す。
    
    - 副作用を排除する。必要な値はuseEffectで
    

- 不変性(immutability): 1度設定した値は書き換えない。
