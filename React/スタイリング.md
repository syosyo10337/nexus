# 1. インラインスタイル

styleプロパティにオブジェクト(ここではstyle)プロパティを渡すことで、成立させる。  
//まるでCSSプロパティの記述をしているように、記述でき、反映させてくれる。  
//ただし、プロパティ名に当たる部分はlowerCamelで書くこと。  
`

```TypeScript
const style = {
        width: 120, //単位を省略でき、その場合にはpxとして認識される。
        height: 60,
        display: "block",
        fontWeight: "bold",
        borderRadius: 9999,
        cursor: "pointer",
        border: "none",
        margin: "auto",
				background: isSelected ? "pink" : "", 
				//あくまでJSのオブジェクトなので、三項演算子を使って
				//また、""の場合には、そのCSSプロパティの値がないと評価されます。
};
	return (
			<>
				<button onClick={clickHandler} 
								style={style}>ボタン
				</button>
				//propsのプロパティstyleに対して直接オブジェクトを渡す際には、{}を二重にする。
				<div style={{ textAlign: "center"}}>
					{isSelected && "クリックされました。"}
				</div>
			</>
	)
```

## インラインスタイルの欠点

1. コードの再利用性が悪くなる。

2. CSSの擬似要素(::before)や擬似セレクター(::hover)を使用することができない。

3. メディアクエリ @media (min-witdh: 600px){} を使うことができない。

4. パフォーマンスの観点からもあまりよくありません。(タグが読み込まれるタイミングでスタイルを計算するため。）

- >本番環境では使わない。とりあえずスタイルつけられるだけ。

# 2. 外部CSSファイルを読み込む

importの記述に続いて、外部ファイルにCSSを記述すること。また、classNameプロパティを用いてクラスの付与によって、実現させる。さらに、JSX内で条件分岐させることで、クラスの付け外しもできる。

```TypeScript
import "./Example.css";

<button>
	className={`btn ${isSelected ? "selected" : ""}`} 
	onClick={clickHandler}>
		ボタン
</button>
```

//またテンプレートリテラルを使って必要に応じて式展開付きの文字列を操作することもできる。この場合には外側の{}がJSXの記述を認識させる記述で、``に囲まれた${}の部分はテンプレートリテラルの表記であることも忘れずに

## 外部CSSファイルの読み込みの欠点

- コンポーネントごとにスタイルを当てることができない。  
    最後に読み込まれたCSSファイルが上書きされるので、コンポーネントの読み込まれる順番でCSSのスタイルも考えなければならない。  
    画面全体にlayoutとしたい場合には、外部ファイルで適応させるのは良いが、  
    コンポーネント毎にスタイルを当てたい場合には得策ではない。クラス名を適宜分ける様な工夫が必要。BEM等を意識しないといけないということになりそうですね。👎

# 3. CSSモジュール

⚠️

**将来的に廃止になるかも、非推奨**

https://github.com/webpack-contrib/css-loader/issues/1050

- 外部にCSSファイルを持ちながらも、importの記述を変更する

e.g.)  
import styles from "./Example.module.css";

- CSSモジュール側は、CSSファイルになっていて /_クラスセレクターや、IDセレクタ_/でスタイルが羅列させる。  
    これを、styles(import部の記述)のプロパティのように、classNameプロパティに記述する。これによって、HTMLでは自動生成されたユニークなクラスネームがついているので、class名の名前衝突を避けられる。

```TypeScript
<button 
	className={`${styles.btn} ${isSelected ? styles.selected : "" }`}
	id={style.attr} //あんまり使わないけどね。
	onClick={clickHandler}>
	ボタン
</button>
```

# 4. CSS in JS

基本的には、ライブラリを使う。有名なのは"styled-components"

自前で用意する時は、インストールする必要がある。

- タグ付きテンプレート  
    `styled.button``;` ()のように関数の呼び出しを意味する。これは、${}テンプレートリテラル内で関数を解析できる。

cf. [https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Template_literals](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Template_literals)

オブジェクト形式でも書くことができるらしいよ。

e.g. ``の中で${}に対して、propsのプロパティを受け取りながら、動的にスタイルを書き換える。

```TypeScript
const StyledButton = styled.button`
	margin: auto;
	border-radius: 9999px;
	border: none;
	display: block;
	width: 120px;
	height: 60px;
	font-weight: bold;
	cursor: pointer;
	//${}の中に、(props) => の形で関数を利用できる。
	background: ${({isSelected}) => isSelected.isSelected ? 'pink' : ''}
`; 
```

## スタイルの継承

新たにコンポーネントを作成しながら、`styled(/`_`継承したいクラスを記述`_`/)`  
その後続いて、``にて、オーバーライドしていきたい。プロパティを記述する。

```TypeScript
const OrangeButton = styled(StyledButton)`
  background-color: orange;
  :hover {
    color: red;
  }
`;
```

同様にして、@mediaクエリ等も使用することができる。  
さらにネスト記法でScssっぽくなるを適用することもできる。  
自動で一意にクラスが適応されるため、影響がされなくて良い。

## 中でもNext.jsだと

以下のような書き方ができるらしい。

```Shell
import React from "react";

const Component: React.FC<React.ComponentProps<"h2">> = (props) => {
  return (
    <>
      <h1 {...props} className="policy-md-components-heading2">
        {props.children}
      </h1>
      <style jsx>{`
        .policy-md-components-heading2 {
          /* Headingリンクを踏んだときにAppBarを考慮した位置にスクロールするようにするトリック */
          /* padding-top: AppBarHeight + 8px = 44 + 8 = 52px */
          /* margin-top: -AppBarHeight = -44px */
          padding-top: 52px;
          margin-top: -44px;
          padding-bottom: 8px;
          font-weight: 500;
        }
      `}</style>
    </>
  );
};

export default Component;

```
