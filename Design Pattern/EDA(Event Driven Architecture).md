# Eventとは

状態が変更されたことをしめすシグナル。

特徴としては、

- 送信先へのの変更の指示は行わない。→ 関心の分離、疎結合化

- 実行結果を受け取らない。　→　非同期処理への適応

- イミュータブル　→ コピーすることで、容易にスケールする。

概念として理解しておきたいこと、

ユーザからのアクションと、ダウンストリームの処理は同期的に処理を行う必要があるのか？

非同期化することによって、

応答性の改善と、依存性の削減が実現できる

# 連携処理をするコンポーネントが必要だね？

# イベント駆動の連携要素について

## event router

event producer と event consumerを抽象化することでルーティン構築を用意にする。

適切なイベントを適切なサービスにルーティングするための、filtering

Amazon SNSは、snsトピックをつかった、pub/sub型でのイベントルーターを構築します。

## event store

イベントが処理できるまでbufferingしておく。　

Amazon SQS

Kinesis Data steams
