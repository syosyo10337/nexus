## 個人レベルでの設計時のかける時間について
ch.19 p62
個人で設計するレベルの物であれば1h時間以上かけて変更するような内容は危険信号として捉えるべき

### コードの変更は一部に集中する
パレートの法則のように,20%のコードが80%の機能を担っている。
コードの変更はその20%のコードに集中するので、毎回数時間を使う構造的変更が起きることは理論上少ないはず。
そのため、1h以上かかる場合には黄色信号と言うことらしい。

**私の私見: この部分について、もしコードの品質がとても悪い場合には必ずしも適用できないと思うが、、この場合でも影響範囲を考えて、boundaryを決めるようにする。
