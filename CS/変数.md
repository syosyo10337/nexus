## **変数**（variable）**とは**　

数値や文字列などのデータを保持するための領域。変数名(label)を付けることでいつでも取り出すことができる。

変数は宣言、代入、参照という 3 つの主な操作が行われます

## メモリとアドレス

---

### メモリ

コンピュータの中でデータを保持している場所の 1 つに、**メモリ**（memory）があります。メモリはセルと呼ばれる小さな領域に分割され管理されています。1 つのセルには 1 バイト（8 ビット）のサイズが割り当てられています。

[![](CS/impoted/Attachments/Screen_Shot_2022-10-31_at_14.44.04.png)](%E5%A4%89%E6%95%B0/Screen_Shot_2022-10-31_at_14.44.04.png)

### メモリアドレス

1 つ 1 つのセルの位置を**メモリアドレス**（memory address）といいます.

CPUはメモリアドレスをもとに格納されているデータを参照、取得する。

プログラミング言語によっては、16進数でメモリアドレスを取得することができる. メモリアドレスは通常、2進数では桁数が多くなる為、16進数で表現される。

## コンパイラ(Compiler)

私たちの言語（高級言語(普段書いているプログラミング言語)）からコンピュータが理解できる機械語へ翻訳するプログラムのこと。この方式を用いて記述される言語はコンパイラ式言語と呼ばれ, C++/Javaなどがあげられる.コンパイル前のプログラムはソースコードと呼ばれる

## 変数の宣言

---

プログラムでどのような名前の変数を用いるか予め示すことを**宣言**（declaration）といいます。

### ****静的型付け言語（statically typed language）****

```TypeScript
//e.g.

int wheelNumbers;
// wheelNumbers という int 型の変数を宣言します
// この時、コンピュータはプログラムの "実行前" に、wheelNumbers にメモリアドレスを割り当てます
```

### ****動的型付け言語（dynamically typed language）****

```JavaScript
wheelNumbers;
// wheelNumbers という変数を宣言します
// コンピュータはプログラムの "実行時" に、wheelNumbers にメモリアドレスを割り当てると同時にデータ型も決定します
// したがって、wheelNumbers は、整数、文字列、ブーリアン値のようなデータ型になる可能性があります
```

簡単に実装できますが、プログラムを実行しないとバグが分からないという難点を持っています。

## 変数の代入

---

宣言した変数に対して、データを関連付ける操作のこと。

多くのプログラミング言語では、宣言と代入を同時に行うことができます。宣言された変数に初めて代入を行う場合、**初期化と呼ばれます**。代入には代入 (=) 演算子が用いられます。

cf)Rubyにおいては、変数の宣言だけを行うことはできず、必ず代入もセットで、初期化する必要がある。

### 静的型付けの場合

```Java
//Java

//変数の宣言と代入
char letter = 'a';
```

### 動的型付けの場合

```JavaScript
//JavaScript

//変数の宣言と代入 -> 初期化
let letter = "a";
```

## 型変換（キャスト）type casting

### 暗黙の型変換

コンパイラが自動的に型変換を行います。代入時と計算時に暗黙の型変換が行われます

- 代入時の変換

代入する変数の型に変換します。言語によって異なります。

- 計算時の変換：

計算時、一番精度の高い型に変換します。
