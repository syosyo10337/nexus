 

# インデックスを貼るということ

# 1. データ構造の追加

インデックスを貼ると、データベースはB-tree（または他の構造）という追加のデータ構造を作成します。

### インデックスなしの場合（フルテーブルスキャン）

```SQL
SELECT * FROM viewed_items WHERE user_id = 123;
→ テーブルの全行を1行ずつ確認 → O(n)
```

### インデックスありの場合

```SQL
→ B-treeを辿って該当行を特定 → O(log n)
```

### 2. トレードオフ

|   |   |   |
|---|---|---|
|側面|pros|cons|
|READ|検索が高速化( O(n) → O(log n))|-|
|WRITE|-|INSERT/UPDATE/DELETEが遅くなる。 インデックスの更新が必要になるため|
|storage|-|追加の容量を圧迫する|
