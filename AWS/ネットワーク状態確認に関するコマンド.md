 

# ネットワーク状態確認に関するコマンド

- ping

```Bash
	$ ping 宛先ホスト名or宛先IPアドレス
```

手元の環境からAWS上のwebサーバーに到達できるかや、webサーバーDB

サーバー間で通信ができるかを調べることができる。Ctrl + c で終了

応答速度が見れます。（ICMPプロトコルを使っているインターネット層）

- traceroute

```Bash
$ traceroute URL/
```

こちらもICMPを使ったコマンド。宛先へのネットワーク上の経路を確認するとkに利用するコマンドです。*表示されるのは、ICMPを受け付けない構成になっていることを示す。

ルーティングを設定しているが、予想以上に時間がかかっているときなど経路確認的な意味合いで用いると良い。

- telnet

```Bash
$ telnet 宛先ホスト名or宛先IPアドレス　ポート番号
#port番号を省略すると、telnetコマンドのwellknownポート23番が使われる。
```

TCP到達性の確認、つまりpingなどで疎通確認はできたものの、WEBサーバーのミドルウェアの起動角煮にゃ、ファイアウォールなどでのフィルタリングがされていないかなど、特定のポートへの到達確認するために使える。

### プログラムの待ち受けポートを確認する

list of open file コマンドを使用する。

[LISTEN] と書かれているのが、他のコンピュータから通信を待ち受けてしているポートを表し、

[ESTABLISHED]が相手と通信中のポートを表す。

```Bash
$ sudo lsof -i -n
# -i 「-i」で全てのネットワークソケットを対象にする。「-i4」でIPv4、「-i6」でIPv6を指定できる。この他、「-iTCP」「-iUDP」「-i@ホスト」「-i:ポート番号」を指定可能。
# -p <プロせスID>　表示対象にしたいプロセスIDを指定する
# -n　ホスト名の換わりにIPアドレスを表示する
# -P ポート名の換わりにポート番号を表示する
```
