# Docker Image とは

---

Dockerfileによって作成される特定の**"環境のスナップショット"です。**。（環境を再現する機能。テンプレートやアーカイブのようなイメージ

例えばCentOSやUbuntuやAlpineなどのOS、NginxやMySQLのようなソフトウェア、PHPやRubyのようなランタイムなどの環境を提供してくれます。

e.g.)

```Bash
$ docker pull ubuntu #docker hubからローカルにイメージを取得
$ docker run ubuntu #イメージの起動

#ubuntuのイメージで、bashを起動
$ docker run -i -t ubuntu bash

#-i コンテナの標準入力を有効化(ubuntuのbashで入力を受け付ける)
#-t ttyを有効化(接続端末（接続している人の端末）のデバイスファイル名を出力
#オプションを連結することも可能
$ docker run -it ubuntu bash
```

✏️

取得していないDocker Imageを `docker run`で起動しようとすると、自動で`docker pull <イメージ名>`をしてDockerHubへ取得しに行きます。  
(毎回docker pull することは少ないが、runと同時にpullも実行されていることは意識しましょう。)

## タグを付与して、バージョンを指定する。

```Bash
ex)
$ docker run python:2.7 python --version
#タグを指定しない場合のデフォルト値は:latestで、最新版を取得する

#-> 2.7でるはず
```

Tagの命名規則的なもの

1. Docker公式のイメージ
    
    - `<イメージ名>:<タグ>`
    

2. ユーザーが作成したイメージ
    
    - `<ユーザー名>/<イメージ名>:<タグ>`
    

3. 非公式レジストリのイメージ
    
    - `<レジストリ名>/<ユーザー名>/<イメージ名>:<タグ>`
    

## Dockerfileからイメージをビルドする

---

Dockerfileはあくまで、イメージを構築するための手順を記述したファイルに過ぎず、Dockerfileがイメージになるわけではない。

つまり、DockerはDockerfileを頼りにDocker imageを作成する、このことを**Dockerイメージをビルドする**と表現する。そして構築されたイメージはコンテナ実行時に利用される。

docker image buildを行う際には、FROMで指定したイメージを一度レジストリからダウンロードし、それをベースイメージとして新たにイメージをビルドしていますが、一度取得したDocker イメージは削除しない限り、ホスト内に残り続けます。`—pull=true`オプションを指定した場合には、FROMに指定したイメージを強制的にレジストリから取得します。

## ローカルにあるキャッシュから差分ビルドする

---

もし、ローカル上にベースイメージのキャッシュがある場合には、dcokerはimage をbuildする際に、差分を活かして効率的にビルドしようとするが、レジストリのlatestがキャッシュした後に更新されている可能性ものあるので、その際は、前述の`—pull=true`をオプションに指定する

## Docker imageのタグとバージョン

---

docker imageにつけられるタグ(:latest)などは、特定のバージョンのイメージにつける識別子です。具体的にバージョンとは、イメージIDのことです。イメージIDは、アプリの変更（Dockerfileの編集や　COPY元を変更）などするたびに更新される一意はIDで、これをバージョンを広くは呼んでいます。

そのため、:latest タグは、時系列的に最新のものを表しているのですね。

### optionメモ

- `-t`: 使用するtagの指定

- `-f`: name of the dockerfile
