# テストコードの書き方
## テストの種類
- 単体テスト(ユニットテスト)
	関数やコンポーネント単位で行うテスト
	e.g.) Jest, @testing-library

- 結合テスト(e2eテスト)
	e.g.)Selenium、Cypress
etc


# @testing-library
さまざまなテストライブラリを含むパッケージです。
```
- @testing-library/react
Reactのコンポーネントのレンダリングなどに使用
- @testing-library/dom
DOMのテストに使用
- @testing-library/jest-dom
JestのDOMのテストで使用
- @testing-library/user-event
UIイベントの実行に使用
```

# Jestとは
複雑な設定の必要がないテストフレームワークのこと。
- 初期設定不要
- テスト用関数が利用可能
	(e.g.test, expect, toBe)
- 非同期処理のテスト
- 外部APIのmock化
などができる。

# テストコードの基本
-----
Arrage:  準備(テストデータ、条件、環境の設定).
Act: 実行(ロジックの実行、関数の実行).
Assertion: 結果の確認(実行結果と期待する結果の比較).
-----



e.g.)//カウントアップするボタンのテスト

import { render, screen, fireEvent } from "@testing-library/react";
import Counter from "./Counter";


test('if countUp_+1 works properly', () => {
  // Arrange
	render( <Counter originCount={0} />);
	
	// 更新前
const spanElBeforeUpdate = screen.getByText('現在のカウント:0')
expect(spanElBeforeUpdate).toBeInTheDocument();

  // Act
  const btn = screen.getByRole('button', {name: 'カウントアップ'})
  fireEvent.click(btn);

  // Assertion
  const spanEl = screen.getByText('現在のカウント:1')
  expect(spanEl).toBeInTheDocument();
});



### fireEvent関数
この関数のメソッド(e.g. click()など)を使用すると、その要素に設定されたイベントを発火させることができる。

e.g.)
//カウントアップというテキストの入ったbtnが持つonClickイベントを発火させている例。
const btn = screen.getAllByRole('button', {name: 'カウントアップ'})
fireEvent.click(btn)


## テストケースをまとめる。
- describe関数を使う
describeはネストできる。構造だけ取ると、
e.g.)
describe('Counterコンポーネントの動作確認', () => {

  describe('初期表示の正常確認', () => {
    test('現在のカウントが0かどうか', () => {
      render(<Counter originCount={0} />);
  
      const spanElBeforeUpdate = screen.getByText('現在のカウント:0')
      expect(spanElBeforeUpdate).toBeInTheDocument();
    });

    test('現在のカウントが1かどうか', () => {
      render(<Counter originCount={1} />);
  
      const spanElBeforeUpdate = screen.getByText('現在のカウント:1')
      expect(spanElBeforeUpdate).toBeInTheDocument();
    })
  }); 

  describe('ボタン制御の確認', () => {
    test('「カウントアップ」押下で「現在のカウント」が+1されるか', () => {
      render(<Counter originCount={0} />);
      
      const spanElBeforeUpdate = screen.getByText('現在のカウント:0')
      expect(spanElBeforeUpdate).toBeInTheDocument();
    
      const btn = screen.getByRole('button', { name: 'カウントアップ'});
      fireEvent.click(btn);
    
      const spanEl = screen.getByText('現在のカウント:1')
      expect(spanEl).toBeInTheDocument();
    })
  
    test('「カウントダウン」押下で「現在のカウント」が-1されるか', () => {
      render(<Counter originCount={0} />);
      
      const spanElBeforeUpdate = screen.getByText('現在のカウント:0')
      expect(spanElBeforeUpdate).toBeInTheDocument();
    
      const btn = screen.getByRole('button', { name: 'カウントダウン'});
      fireEvent.click(btn);
    
      const spanEl = screen.getByText('現在のカウント:-1')
      expect(spanEl).toBeInTheDocument();
    })
  
    test('「クリア」押下で「現在のカウント」が0になるか', () => {
      render(<Counter originCount={2} />);
      
      const spanElBeforeUpdate = screen.getByText('現在のカウント:2')
      expect(spanElBeforeUpdate).toBeInTheDocument();
    
      const btn = screen.getByRole('button', { name: 'クリア'});
      fireEvent.click(btn);
    
      const spanEl = screen.getByText('現在のカウント:0')
      expect(spanEl).toBeInTheDocument();
    }) 
  });
});

# userEventを使う。
script(コード上で)、ユーザの操作を擬似的に表現する時に使用する。
//fireEvent(dispatchEventを使用している。)に似ている.
userEventの方がより、純粋にユーザの画面操作を再現することになる。


//基本書式。準備
import userEvent from "@testing-library/user-event";

const user = userEvent.setup();

e.g.)
import { render, screen } from "@testing-library/react";
import Message from "./Message";
import userEvent from "@testing-library/user-event";

describe("Messageコンポーネントの動作確認", () => {
  describe("初期表示の正常性確認", () => {
    test('入力された文字が画面上に存在', () => {
      render(<Message />);
      const h2El = screen.getByRole("heading", {name: "入力された文字:" })
      expect(h2El).toBeInTheDocument();
    });
  });

  describe("画面操作正常性確認",  () => {
    test('入力値変更 -> 送信ボタン押下', async () => {
      const user = userEvent.setup();
      render(<Message />);
      const inputEl = screen.getByRole('textbox');
      const btnEl = screen.getByRole('button');
      const h2El = screen.getByRole("heading", {name: "入力された文字:" })
      //初期状態を確認
        //ボタンが非活性であることを主張
      expect(btnEl).toBeDisabled();
        //input要素が空であることを主張
      expect(inputEl.value).toBe("");

      await user.type(inputEl, "こんにちは");
      expect(btnEl).not.toBeDisabled();
      
      await user.click(btnEl);
      expect(h2El.textContent).toBe("入力された文字:こんにちは")

    });
  });
})

## vueインスタンスを使用する。
ここでの vm は Vue.js のコンポーネントインスタンスを指しています。Vue.js では、コンポーネントインスタンスを "ViewModel" と呼びます。(これは Model-View-ViewModel（MVVM）パターンに由来します。)

Vue.js の公式ドキュメントによれば、vm は "a shorthand for ViewModel"（ViewModelの略語）です。Vue.js では、コンポーネントインスタンスはデータ、メソッド、コンピューテッドプロパティ、ウォッチャーなど、コンポーネントの全ての振る舞いと状態を管理します。

したがって、このコードでは lazy('component').vm.getArrivalLabel(1) は、ProductSearchLink コンポーネントインスタンスの getArrivalLabel メソッドを呼び出しています。このメソッドは恐らく、特定の数値（この場合は 1）に対応するラベルを返す役割を持っているでしょう。
