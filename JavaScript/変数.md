### `let`は再宣言不可能な宣言

letは初期値なしでの変数宣言もできる。

```JavaScript
//変数の宣言（と代入）
let myVariable = 'value'; 
//var も変数定義に使えるが、古い書式+問題があるらしい。
//letは初期値なしでの変数宣言もできる。
let x; //-> undefined

//変数の参照.変数名を記述する
myVariable; //->"value"


//変数(constant)の宣言（と代入）
//変数と異なり、値の書き換えができない。
const myVariable = 'value';

```

### `const`の変数宣言

初期値は必須です。constキーワードは、再宣言と再代入が==不可能==な宣言

```Bash
const y = 2;
```

⚠️

**constは可変オブジェクトを保護しない  
**constでは再代入不可な変数を宣言しているだけで、ミュータブルなオブジェクトのプロパティ(オブジェクトのプロパティや、配列の要素)についても不変にするような保護効果はない。

e.g.)オブジェクトの要素の例

```TypeScript
const obj = { a: 1 };
obj = { a: 2 }; // 再代入は不可
obj.a = 2; // プロパティの変更はできる
```

e.g.)配列の要素の場合

```TypeScript
const arr = [1, 2];
arr = [3, 4]; // 再代入は不可
arr.push(3); // 要素の変更はできる
```

☝

TypeScriptを用いて、配列やオブジェクトを不変にするためには`readonly`に変更する必要がある。

# 変数の命名規則(JavaScript)

---

英数字やアンダースコア[0-9a-zA-Z_]のみを用いて、数字で始めない(_でも始めない)。ハイフン(-)は使えない。

lowerCamelCaseを用いる。e.g.)`oddNumber`

JavaScriptでは大文字小文字が区別されるので、mynameとmyNameは別の変数として扱われます。

## varの問題点

---

### 同名の変数宣言

`var`の変数宣言では同じ変数名で宣言をした場合にエラーとならずに、後から宣言された変数が有効となります。つまり、変数の上書きができちゃう。

### **グローバル変数の上書き**[](https://typescriptbook.jp/reference/values-types-variables/vars-problems#%E3%82%B0%E3%83%AD%E3%83%BC%E3%83%90%E3%83%AB%E5%A4%89%E6%95%B0%E3%81%AE%E4%B8%8A%E6%9B%B8%E3%81%8D)

`var`はグローバル変数として定義されたときに、`window`オブジェクトのプロパティとして定義されるため、既存のプロパティを上書きする危険性があります。

たとえば、ブラウザ上で`innerWidth`変数をグローバル変数として定義してしまうと、標準APIの`window.innerWidth`が上書きされるため、ブラウザの幅を変更しても常に同じ値が返ってくるようになってしまいます。
