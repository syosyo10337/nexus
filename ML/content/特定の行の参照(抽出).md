## `df['column_name'] < n`

などのように、データフレームの特定のカラムに対して比較演算子を使うと、Series型のbool値を持つ値が返ってくる

```Python
e.g.)
print(test['Acolumn'] < 2 )
#->
0     True
1    False
2    False
```

### ==データフレームの特徴==

df[XXX]のXXXにあたる部分に対して、bool型の値を含むSeriesまたはリストを指定すると、==Trueである行を抽出することができる。==

つまり、 Series内で　Trueとなっているインデックスと同じインデックスとなるデータフレームのrowを抜き出していることになる。

## `df[df[’col_name’] < condition_val]`

condition_valの比較条件を満たす行を抽出する。

## 複数条件の指定

- 必ず、bool値を持つSeriesになる部分を()でまとめる。

- 論理演算子としてのand orは使えない(& |)を使用する。

```Python
e.g.)
df2[(df2['SNS2'] > 1000) & (df2['sales'] < 8500)]
```

# (2)参照

## `pandas.DataFrame.loc[index, col_name]`

特定の行と列の取りだしに使える。行番号とカラム名をしてすることで、特定の値だけを取り出すことができる。

どちらもリストも取ることができ、その場合には、複数の行や列を参照することができる。

```Python
e.g.)
df3.loc[2, 'SNS1']

index = [2,4,6]
col_name = ['SNS1', 'actor']
df3.loc[index, col_name]
#->
SNS1	actor
2	158.0	6340.388534
4	209.0	10908.539550
6	153.0	7237.639848
```

### ==Slice構文を使う場合の注意点==

python構文のスライスと異なり、start/stopの両方の値を含む。[A:B] の時A以上B==以下==になる。

- “ : “ だと全ての範囲

- “A: “だと、A以上すべて

- “ : B”だと、B以下すべて(B含めた、Bより右のカラム)

よって、==特徴量(カラム名、インデックス、データを含むdf)を抽出する際==には、

```Python
X = df3.loc[ : , 'SNS1':'original'] 
y = df3['sales']

と指定する。

より実践的にいうと、where句にあたるような複数条件で絞り込んだ indexとbool値のSeriesの条件を、indexの値と捉えて指定することもできる

```
