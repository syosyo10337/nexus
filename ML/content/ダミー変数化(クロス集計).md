 

# ダミー変数化(クロス集計)

---

sk-learnでは、数値以外の特徴量を取ることができない。

そのため、性別や、成否や種別等の文字列を数値に仮に変換することを**ダミー変数化, ホットエンコーディング**と呼ぶ

```Python
e.g.)
# 'female' -> 0 'male' -> 1にダミー変数化する例
df2.loc[df2['Sex'] == 'male', 'Sex'] = 1
```

実際には、それ専用の関数も用意されている

## `pd.get_dummies(data,` `_**drop_first=False**_``**,**``)`

```Python

male = pd.get_dummies(df2['Sex'], drop_first =True)
male
```

上の例では、一番最初のレコードのSexカラムの値’male’を新たなカラム名とした0/1(maleのデータに対しては1)とした**データフレームを返す**

## optionの`drop_first`について

これは、

> **drop_firstbool, default False**
> 
> Whether to get k-1 dummies out of k categorical levels by removing the first level.
> 
> (pandas docsより)

たとえば３種類のものをダミー変数化する時にカラムを3つ作って、該当するものを1とするか(drop_first=False(default))

(**3 - 1)** つのカラムを作って、どのカラムでも0になった時、最後の一種類であると理解する分け方を示す。

## 多重共線性

drop_firstをfalseにした時に、多重共線性を持ち、それによって、ダミー変数のカラムによる影響を受けやすいモデルになるそう。

# label encording

例えば、色カラムにて、赤/青/緑という値を0/1/2というようにしてダミー変数として数値化することはできる。

がしかし、機会学習の文脈において 青(1)+ 青(1) → 緑(2)となるのは、予測性能に影響するので避けた方が良いそう。
