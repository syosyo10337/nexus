決定木モデルは、正解ラベルのどの値になるのかを予測するための**フローチャートを作成する**

決定木の秀でている点としては、分析結果が直観的に非常に理解しやすいフローチャートで表せる。

この手法は、親ノードから、子ノードの不純度(ジニ不純度)がどのように変化したかを分析し、それを元に最適な条件分岐を設定していく。

- 不純度
    
    ある条件において、２分されたデータ群の目的変数の種類の比率のこと。
    
    例えば、あるとき、２分されたデータの正解ラベルが
    
    {1, 1, 0, 0} {1, 0, 0, 0}に割れた場合には、左側が不純度が高い(1:1)というように評価する指標のこと。

# `tree.DecisionTreeClassifier`

sklearnでは、このモジュールを用いて決定木分類を行う

## 乱数と再現性

決定木モデルだけでなく、==機械学習のモデル作成において、乱数を用いてモデルを作成するため、同一データによってモデルを２度生成した場合においても、異なる予測をするモデルが完成する可能性があることを意味する。==

## 再現性の確保

randam_stateの値を指定することで、生成される乱数をrandam_stateの値ごとに固定できる。

```Python
e.g.)

from sklearn import tree
model = tree.DecisionTreeClassifier(\
	max_depth = 2, \
	random_state = 0 \
)
```

### max_depth

何回層まで、決定木を深くするかを指定できる。

==このように人間の塩梅で決められるパラメータをハイパーパラメータという==　一般に深いほど、試行回数が増えるため、細かい分類を行うことができる。しかし、デメリットを挙げるとすればフローチャートとした時の視覚的な理解のしやすさが低下することがあげられる。

さらに、[==過学習==](../%E3%82%BF%E3%82%A4%E3%82%BF%E3%83%8B%E3%83%83%E3%82%AF%E5%8F%B7%E3%81%AE%E7%94%9F%E5%AD%98%E8%80%85\(%E5%88%86%E9%A1%9E\)/%E9%81%8E%E5%AD%A6%E7%BF%92%2038213a5c2841478c89e5a1a3df7b2392.html)と呼ばれ、むしろ精度を下げることもある。

### radom_state

シード値の値を計算する。この時取りうる値は0以上の整数になる。必ず乱数シードの固定を行うことを忘れないで。

### class_weight

正解データの割合が不均衡の場合でも  
分岐条件を考える際に、比率の大きいデータの影響を小さく、小さいデータの影響をおおきくし、分岐条件の決定をbalancedさせることで、予測性能を向上させる効果が期待できる。  
model.fit(x_train, y_train)
