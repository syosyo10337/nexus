# 概要

## 複合フラグラメント

以下のような種類があります

- alt(条件分岐)

- assert(validation的な)

- par(並列)

- consider(重要)

- neg(不正パターン)

- ignore

- opt(オプショナル)

- ref(外部参照）

- loop

- break

- critical(排他処理)

# in Mermaid

ここでは、mermaidでの書き方も追加します。

## 基本構成要素

### 1. ライフライン（参加者）の定義

```Mermaid
equenceDiagram
    participant User
    participant System
    participant Database as DB
```

### 2. メッセージの種類

**同期メッセージ（実線矢印）:** `**->>**`

```Mermaid
sequenceDiagram
    participant A
    participant B
    A->>B: 同期メッセージ
```

**非同期メッセージ（実線矢印、矢尻なし）:** `-)`

```Mermaid
sequenceDiagram
    participant A
    participant B
    A-)B: 非同期メッセージ
```

**応答メッセージ（破線矢印）:** `**-->>**`

```Mermaid
sequenceDiagram
    participant A
    participant B
    A->>B: リクエスト
    B-->>A: レスポン
```

### 3. 実行仕様（アクティベーション）: `activate/deactivate`

```Mermaid
sequenceDiagram 
    participant A
    participant B
    A->>B: メッセージ
    activate B
    B-->>A: 応答
    deactivate B
```

## 複合フラグメント

### 1. 条件分岐（Alt） : `alt/else/`

```Mermaid
sequenceDiagram
    participant User
    participant System
    
    User->>System: 課金リクエスト
    alt 15歳未満
        System-->>User: 上限1,000円まで
    else 15歳以上20歳未満
        System-->>User: 上限1万円まで
    else 20歳以上
        System-->>User: 上限なし
    end
```

### 2. 条件指定（Opt）: `opt/`

```Mermaid
sequenceDiagram
    participant Customer
    participant CashRegister
    participant TaxFreeSystem
    
    Customer->>CashRegister: 商品購入
    opt 外国人の場合
        CashRegister->>TaxFreeSystem: 免税処理
        TaxFreeSystem-->>CashRegister: 処理完了
    end
    CashRegister-->>Customer: 会計完了
```

### 3. ループ処理（Loop）: `loop`

```Mermaid
sequenceDiagram
    participant CashRegister
    participant ItemDB
    
    loop 商品数分繰り返し
        CashRegister->>ItemDB: 商品価格照会
        ItemDB-->>CashRegister: 価格返却
    end
```

### 4. 並列処理（Par）: `par`

```Mermaid
sequenceDiagram
    participant ECSystem
    participant StockSystem
    participant AccountingSystem
    
    ECSystem->>ECSystem: 販売確定
    par 在庫管理
        ECSystem->>StockSystem: 在庫更新
        StockSystem-->>ECSystem: 更新完了
    and 会計処理
        ECSystem->>AccountingSystem: 売上記録
        AccountingSystem-->>ECSystem: 記録完了
    end
```

### 5. 中断処理（Break）: `break`

```Mermaid
sequenceDiagram
    participant System
    participant StockDB
    
    System->>StockDB: 在庫確認
    StockDB-->>System: 在庫数: 0
    
    break 在庫切れの場合
        System-->>System: エラーメッセージ表示
    end
```

### ノート（注釈）

```Mermaid
sequenceDiagram
    participant A
    participant B
    
    A->>B: メッセージ
    Note over A,B: この処理は重要です
    Note right of B: Bでの処理
    B-->>A: 応答
```

### 自己呼び出し

activateを活用する

```Mermaid
sequenceDiagram
    participant System
    
    System->>System: 内部処理
    activate System
    System-->>System: 処理結果
    deactivate System
```

### 表現できない複合フラグメント

- **Assert（アサーション）**: 直接的な表現方法なし → Noteで代用

- **Consider（重要処理）**: 専用記法なし → 色分けやNoteで強調

- **Ignore（無効）**: 専用記法なし → コメントで表現

- **Neg（不正）**: 専用記法なし → alt/elseで否定条件として表現

- **Ref（外部参照）**: 専用記法なし → Noteやリンクで説明

- **Critical（排他処理）**: 直接的な表現なし → 文書化で補完

### 代替表現例

**Assert（アサーション）をNoteで表現**

```Mermaid
sequenceDiagram
    participant Stock as 在庫システム
    participant Check as チェック機能
    
    Stock->>Check: 在庫数確認
    Note over Check: Assert: 現在の在庫数 = 昨日の在庫数 - 本日の出荷数
    Check-->>Stock: 検証結果
```

**外部参照（Ref）をNoteで表現**

```Mermaid
sequenceDiagram
    participant Main as メインシステム
    participant Sub as サブシステム
    
    Main->>Sub: 処理要求
    Note over Sub: 詳細は「決済処理シーケンス図」を参照
    Sub-->>Main: 処理完了
```

## ✅ 推奨事項

1. **シンプルさを保つ**: 1つのシーケンス図は1つのシナリオに集中

2. **命名規則を統一**: 参加者名やメッセージ名に一貫性を持たせる

3. **適切な粒度**: 詳細すぎず、粗すぎない適切なレベルで記述

4. **文書化**: Mermaidで表現できない要素は別途文書化する

これらの記法を使うことで、UMLの標準的なシーケンス図をMermaidで効果的に表現できます。
