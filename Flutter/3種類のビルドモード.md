# 3種類のビルドモード

Flutterには主に3つのビルドモードがあります：

1. Debug Mode（デバッグモード）：
    
    - アプリケーションの開発中に使用されるモードです。
    
    - `flutter run` コマンドを実行すると、デフォルトでデバッグモードが使用されます。
    
    - デバッグモードでは、アサーション（`assert`）が有効になり、エラーの検出に役立ちます。
    
    - ホットリロード（ソースコードの変更をすぐに反映する機能）が利用可能です。
    
    - パフォーマンスは最適化されていませんが、開発には十分な速度です。
    

2. Release Mode（リリースモード）：
    
    - アプリケーションを実際のユーザーに公開する際に使用されるモードです。
    
    - `flutter run --release` または `flutter build` コマンドを使用してリリースビルドを作成します。
    
    - リリースモードでは、アサーションは無効になり、エラーチェックのオーバーヘッドが削減されます。
    
    - Dartコードは最適化されて、高いパフォーマンスを発揮します。
    
    - ただし、ホットリロードは利用できません。
    

3. Profile Mode（プロファイルモード）：
    
    - アプリケーションのパフォーマンスを分析するために使用されるモードです。
    
    - `flutter run --profile` コマンドを使用してプロファイルビルドを実行します。
    
    - プロファイルモードでは、リリースモードと同様にアサーションが無効になり、Dartコードが最適化されます。
    
    - ただし、DevToolsを使用してアプリケーションのパフォーマンスを分析することができます。
    
    - パフォーマンス測定のオーバーヘッドがあるため、リリースビルドよりもわずかに遅くなります。
    

これらのモードは、以下のような定数を使用して判別できます：

- `kDebugMode`：デバッグモードかどうかを示します。

- `kReleaseMode`：リリースモードかどうかを示します。

- `kProfileMode`：プロファイルモードかどうかを示します。

これらの定数は、`foundation` パッケージからインポートできます：

```Dart
import 'package:flutter/foundation.dart';
```

これらの定数を使用して、現在のビルドモードに基づいてコードを条件付きで実行することができます。例えば：

```Dart
if (kDebugMode) {
  // デバッグモードでのみ実行されるコード
} else if (kReleaseMode) {
  // リリースモードでのみ実行されるコード
} else if (kProfileMode) {
  // プロファイルモードでのみ実行されるコード
}
```

このように、Flutterの3つのビルドモードを理解し、適切に使い分けることが重要です。
