# シェルスクリプト


## シェルとは

シェルは、カーネルが提供する機能を操作する際に、OSと対話的に操作するためのインターフェースです。シェルはOSの、特にカーネル部分を包み込んでいることからその名があり、対話機能を提供します。

## プログラムの主要4要素

シェルスクリプトは、以下の4つの要素で構成されます:

- **順次実行**: コマンドを上から順に実行
- **条件分岐**: 条件に応じて処理を分岐
- **繰り返し**: 同じ処理を繰り返す
- **サブルーチン**: 処理をまとめて再利用可能にする（関数）

# シェルスクリプトの作成と実行

## 基本的な作成方法

シェルスクリプトはテキストファイルとして作成します（`vi`や`vim`などのエディタを使用）。

### スクリプトの基本構造

```bash
#!/bin/bash
# 1行目: シェバン（shebang）でシェルを指定
ls
date
# 2行目以降: 実行するコマンドを記述
```

## 実行権限の付与

```bash
# 実行権限を付与
$ chmod u+x lsdate.sh

# 権限を確認
$ ls -l lsdate.sh
-rwxrw-r--. 1 linuxtext linuxtext 21 Jun 6 09:51 lsdate.sh
```

> **注意**: スクリプトを実行するには、実行権限（`x`）が必要です。

## スクリプトの実行

```bash
# カレントディレクトリのスクリプトを実行
$ ./lsdate.sh
Desktop diff2 ls-l-output ls-usr-bin touched-file
cat-output hosts.bak ls-l-output-second lsdate.sh uniq-sample
diff1 hosts.new2 ls-output score
Fri Jun 6 09:55:05 JST 2008
```

> **解説**: `./`はパス指定で、カレントディレクトリ（`.`）にある`lsdate.sh`を実行することを意味します。

# 基本コマンドと変数 

## echoコマンド

引数で与えた文字列を標準出力に出力します。

```bash
echo [オプション] 文字列
```

### 主要オプション

- `-n`: 改行を抑制（通常は自動で改行されます）
- `-e`: エスケープシーケンスを有効化

### 使用例

```bash
$ echo "Hello, World!"
Hello, World!

$ echo -n "改行なし"
改行なし$
```

## 変数の基本操作

> **詳細**: シェル変数と環境変数の詳細は[シェル変数.md](シェル変数.md)を参照してください。

### 変数の代入と参照

```bash
# 代入は「=」、参照は「$」を使用
$ abc=123
$ echo $abc
123
```

> **注意**: `=`の前後にスペースを入れないこと（コマンドとして解釈されてしまいます）

### 配列変数

Bashでは1次元の配列変数が使用できます。

```bash
# 配列への代入
$ abc[0]=123
$ abc[1]=456

# 配列の参照（波括弧で囲む）
$ echo ${abc[0]}
123
$ echo ${abc[1]}
456
```

## readコマンド

標準入力からデータを読み込み、変数に格納します。既存のデータは上書きされます。

```bash
read 変数名
```

> **注意**: 変数名の前に`$`は不要です（参照ではなく代入のため）

### 使用例

```bash
$ echo $abc
123

$ read abc
aaabbbccc

$ echo $abc
aaabbbccc
```

## 引用符（クォート）の種類と使い分け

### シングルクォート（`'`）

囲んだ範囲を**文字列として**そのまま認識します。

```bash
$ echo 'Value is $abc'
Value is $abc
```

### ダブルクォート（`"`）

囲んだ範囲の`$`付き変数を**変数として展開**します。

```bash
$ abc=123
$ echo "Value is $abc"
Value is 123
```

### バッククォート（`` ` ``）

囲まれた範囲を**コマンドとして実行**し、その結果を返します。

```bash
$ XYZ=`date`
$ echo "It is $XYZ now."
It is Thu Mar 20 06:08:14 JST 2017 now.

# 変数への代入を省略する書き方
$ echo "It is `date` now."
It is Thu Mar 20 06:08:14 JST 2017 now.
```

> **現代的な書き方**: バッククォートの代わりに`$(command)`の使用が推奨されます。

# スクリプトの引数処理

## 位置パラメータ（引数）

スクリプト実行時に渡された引数を参照できます。

### 特殊変数

| 変数 | 説明 |
|------|------|
| `$0` | 実行コマンド名（スクリプト名） |
| `$1`, `$2`, `$3`... | 1番目、2番目、3番目...の引数 |
| `$#` | 引数の個数 |
| `$@` | すべての引数（個別に展開） |
| `$*` | すべての引数（1つの文字列として展開） |

### 使用例

```bash
#!/bin/bash
echo "$1:" $1
echo "$2:" $2
echo "$3:" $3
echo "$0:" $0
echo "$#:" $#
```

実行結果:

```bash
$ ./args.sh aaa bbb ccc
$1: aaa
$2: bbb
$3: ccc
$0: ./args.sh
$#: 3
```

## shiftコマンド

引数の順序を左にずらします（`$1`を削除し、`$2`が`$1`になる）。

### 使用例

```bash
#!/bin/bash
echo "$1:" $1
echo "$2:" $2
echo "$3:" $3
shift
echo "$1:" $1
echo "$2:" $2
```

実行結果:

```bash
$ ./argsshift.sh aaa bbb ccc
$1: aaa
$2: bbb
$3: ccc
$1: bbb  # $2だったものが$1に
$2: ccc  # $3だったものが$2に
```

### 動作の説明

- **shift前**: `$1=aaa`, `$2=bbb`, `$3=ccc`
- **shift後**: `$1=bbb`, `$2=ccc`（`aaa`は削除される）

> **ユースケース**: 多数の引数を順次処理する際に便利です。

# エスケープ文字とエスケープシーケンス

## エスケープ文字（`\`）

バックスラッシュ（`\`）は、直後の1文字の扱いを変更します。

### 行の継続

行末にバックスラッシュを付けることで、コマンドを複数行に分割できます。

```bash
$ echo "I am a cat. As yet I have no name."
I am a cat. As yet I have no name.

$ echo "I am a cat. \
> As yet I have no name."
I am a cat. As yet I have no name.
```

> **利点**: 長いコマンドの視認性を向上できます。

### 特殊文字のエスケープ

```bash
# ダブルクォートを出力する方法1: シングルクォートで囲む
$ echo 'Value of ABC is "$ABC".'
Value of ABC is "$ABC".

# ダブルクォートを出力する方法2: バックスラッシュでエスケープ
$ echo "Value of ABC is \"$ABC\"."
Value of ABC is "123".

# $記号をそのまま表示
$ echo "Value of ABC is \$ABC."
Value of ABC is $ABC.
```

## エスケープシーケンス

`echo -e`オプションを使用すると、エスケープシーケンスが有効になります。

### 主要なエスケープシーケンス

| シーケンス | 説明 |
|-----------|------|
| `\n` | 改行 |
| `\t` | タブ |
| `\r` | キャリッジリターン |
| `\\` | バックスラッシュ |
| `\0nnn` | 8進数で文字を指定（例: `\041`は`!`） |

### 使用例

```bash
$ echo -e "I am a cat. \nAs yet I have no name\041"
I am a cat.
As yet I have no name!
```

# sourceコマンド

## 概要

`source`コマンド（または`.`コマンド）は、指定されたファイルを現在のシェル環境で読み込んで実行します。

> **詳細**: 設定ファイルの永続化については[シェル変数.md](シェル変数.md)を参照してください。

## 通常実行とsourceの違い

### 通常実行（`./script.sh`）

サブシェルで実行されるため、スクリプト内の変数は親シェルに影響しません。

```bash
$ cat set.sh
#!/bin/bash
abc=xyz
echo $abc

$ echo $abc
# 何も表示されない（変数が未定義）

$ ./set.sh
xyz
# スクリプト内で設定された変数が出力される

$ echo $abc
# 何も表示されない（変数はサブシェル内でのみ有効）
```

### source実行

現在のシェル環境で実行されるため、変数が保持されます。

```bash
$ source set.sh
xyz

$ echo $abc
xyz
# sourceで読み込んだため、変数が現在のシェルに残る
```

## 主な用途

- 設定ファイル（`.bashrc`, `.bash_profile`など）の変更を即座に反映
- 共通の変数や関数を複数のスクリプトで共有

```bash
# 設定ファイルの変更を即座に反映
$ source ~/.bashrc

# 短縮形
$ . ~/.bashrc
```

# 条件分岐

## if文

条件に応じて処理を分岐します。

### 基本構文

```bash
if [ 条件 ]; then
    処理1
elif [ 条件2 ]; then
    処理2
else
    処理3
fi
```

> **注意**: `[`の後と`]`の前にはスペースが必要です。

### 文字列比較演算子

| 演算子 | 説明 |
|--------|------|
| `a == b` | aとbが等しければ真 |
| `a != b` | aとbが等しくなければ真 |
| `-z 文字列` | 文字列の長さが0なら真 |
| `-n 文字列` | 文字列の長さが0でなければ真 |

### 数値比較演算子

| 演算子 | 説明 | 英語表現 |
|--------|------|----------|
| `a -eq b` | aとbが等しい | equal to |
| `a -ne b` | aとbが等しくない | not equal to |
| `a -ge b` | aがb以上 | greater than or equal to |
| `a -le b` | aがb以下 | less than or equal to |
| `a -gt b` | aがbより大きい | greater than |
| `a -lt b` | aがb未満 | less than |

### ファイル属性の確認演算子

```bash
# 書き方1: testコマンド
if test -d /path/to/dir; then
    echo "ディレクトリです"
fi

# 書き方2: [ ] (推奨)
if [ -f /path/to/file ]; then
    echo "通常ファイルです"
fi
```

| 演算子 | 説明 |
|--------|------|
| `-f ファイル名` | 通常ファイルなら真 |
| `-d ファイル名` | ディレクトリなら真 |
| `-e ファイル名` | ファイルが存在すれば真 |
| `-L ファイル名` | シンボリックリンクなら真 |
| `-r ファイル名` | 読み取り可能なら真 |
| `-w ファイル名` | 書き込み可能なら真 |
| `-x ファイル名` | 実行可能なら真 |
| `-s ファイル名` | サイズが0より大きければ真 |

### 論理演算子

#### AND（論理積）

```bash
# [ ]内で使用
[ 条件A -a 条件B -a 条件C ]

# [ ]を分けて使用（推奨）
[ 条件A ] && [ 条件B ] && [ 条件C ]
```

#### OR（論理和）

```bash
# [ ]内で使用
[ 条件A -o 条件B -o 条件C ]

# [ ]を分けて使用（推奨）
[ 条件A ] || [ 条件B ] || [ 条件C ]
```

## case文

変数の値に応じて処理を分岐します。

### 基本構文

```bash
case 変数 in
    値A)
        処理1
        ;;
    値B|値C)
        処理2
        ;;
    *)
        デフォルト処理
        ;;
esac
```

> **注意**: 各パターンの終わりには`;;`が必要です。パイプ記号（`|`）で複数の値を指定できます。

### 使用例

```bash
#!/bin/bash
read -p "果物を選んでください (apple/banana/orange): " fruit

case $fruit in
    apple)
        echo "りんごを選びました"
        ;;
    banana)
        echo "バナナを選びました"
        ;;
    orange)
        echo "オレンジを選びました"
        ;;
    *)
        echo "不明な果物です"
        ;;
esac
```

# 繰り返し処理

## for文

リストの各要素に対して処理を繰り返します。

### 基本構文

```bash
for 変数 in 値のリスト
do
    処理
done
```

### 使用例

```bash
# 基本的な使い方
$ for i in a b c d
> do
> echo $i
> done
a
b
c
d

# コマンドの実行結果を使用
for i in `ls`
do
    echo "File: $i"
done

# または $(command) 形式（推奨）
for i in $(ls)
do
    echo "File: $i"
done

# 数値の範囲指定
for i in {1..5}
do
    echo "Number: $i"
done
```

## while文

条件が真の間、処理を繰り返します（条件が偽になったら終了）。

### 基本構文

```bash
while [ 条件 ]
do
    処理
done
```

### 使用例

```bash
#!/bin/bash
count=1
while [ $count -le 5 ]
do
    echo "Count: $count"
    count=$((count + 1))
done
```

## until文

条件が偽の間、処理を繰り返します（条件が真になったら終了）。

### 基本構文

```bash
until [ 条件 ]
do
    処理
done
```

> **whileとの違い**: `while`は条件が真の間継続、`until`は条件が真になるまで継続

## select文

ユーザーに選択肢を提示し、数値入力を促します。

### 基本構文

```bash
select 変数 in リスト
do
    処理
done
```

### 使用例

```bash
select name in "apple" "banana" "orange"
do
    echo "You selected $name"
done
```

実行結果:

```
1) apple
2) banana
3) orange
#? 1
You selected apple
```

> **終了方法**: `Ctrl + C`で中止

## 繰り返しの制御

### break

ループを即座に終了します。

### continue

現在の反復をスキップし、次の反復に進みます。

### 使用例

```bash
#!/bin/bash
while true
do
    echo "Continue? (y/n)"
    read input
    case $input in
        n)
            break
            ;;
        y)
            continue
            ;;
        *)
            echo "Please input y or n."
            ;;
    esac
done
```

実行結果:

```bash
$ ./sample.sh
Continue? (y/n)
y
Continue? (y/n)
a
Please input y or n.
Continue? (y/n)
n
$
```

# 関数（サブルーチン）

## 関数の定義

一連の処理をまとめて、再利用可能にします。

### 基本構文

```bash
# 書き方1: functionキーワードを使用
function 関数名 {
    処理
}

# 書き方2: ()を使用（POSIX互換、推奨）
関数名() {
    処理
}
```

### 使用例

```bash
#!/bin/bash

# 関数の定義
greet() {
    echo "Hello, $1!"
}

# 関数の呼び出し
greet "World"
greet "Bash"
```

出力:

```
Hello, World!
Hello, Bash!
```

### 戻り値

```bash
#!/bin/bash

check_file() {
    if [ -f "$1" ]; then
        return 0  # 成功（真）
    else
        return 1  # 失敗（偽）
    fi
}

# 使用例
if check_file "/etc/passwd"; then
    echo "ファイルが存在します"
else
    echo "ファイルが存在しません"
fi
```

> **注意**: `return`は終了ステータス（0-255）のみ返せます。文字列を返す場合は`echo`を使用し、コマンド置換で受け取ります。

### 関数からの値の取得

```bash
#!/bin/bash

get_message() {
    echo "This is a message"
}

# コマンド置換で値を取得
result=$(get_message)
echo "Result: $result"
```

# デバッグ

## shコマンドによるデバッグ

`sh -x`オプションを使用すると、コマンドや変数の内容を表示しながらスクリプトを実行できます。

### 基本構文

```bash
sh -x スクリプト名
# または
bash -x スクリプト名
```

### 実行例

```bash
$ sh -x ./sample.sh
+ true
+ echo "Continue? (y/n)"
Continue? (y/n)
+ read input
y
+ case $input in
+ continue
+ true
+ echo "Continue? (y/n)"
Continue? (y/n)
+ read input
n
+ case $input in
+ break
```

> **解説**: `+`記号の後に実際に実行されるコマンドが表示されます。

## スクリプト内でのデバッグモード

```bash
#!/bin/bash

# デバッグモードを有効化
set -x

# 処理
echo "Debug mode is on"

# デバッグモードを無効化
set +x

echo "Debug mode is off"
```

## その他の便利なオプション

| オプション | 説明 |
|-----------|------|
| `set -e` | エラーが発生したら即座にスクリプトを終了 |
| `set -u` | 未定義の変数を参照したらエラー |
| `set -o pipefail` | パイプライン内のコマンドが失敗したらエラー |

### 推奨される設定

```bash
#!/bin/bash

# 厳格モード（エラーを見逃さない）
set -euo pipefail

# 以降の処理
```
