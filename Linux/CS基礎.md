ex)

-整数型(Integer)/int/long型
-浮動小数点型/float/double型
-文字型(Character)/char型
-文字列型(String)/str型


*Boolean型について
//JavaScriptでは、typeof とすることでデータ型を調べられる

console.log(typeof false);


nビットは,2n通り(の0/1)の組み合わせがあります。

**int型について --一般的に、整数型は4Byte、つまり32bitの長さがあります。つまり、32bitは 4,294,967,296（= 2**32）通りの2進数の表現が可能.(Integer=整数)
通常の整数は、符号付き整数型（signed integer data type）で、-2,147,483,648 ～ 2,147,483,647を扱える。これとt対照的にunsigned integer data typeもある。

int型の上限がある言語のない言語がある。Rubyはない。Java(32b)/PHP(64b)は存在する。

**大きな整数型(long型) --int型に上限があるものは、より大きい数字は別の型がある。
ex)inJava int(4B) -> long(8B) 2倍の数を扱える

16進数（hexadecimal) --0xFEというように0xをつけて10進数と区別する。16進数は、一桁4bit


*char型 --文字コードに沿って、2進数と文字を対応させている
unicode ---世界中の文字を表現できるように、 1つの文字に対して符号位置（コードポイント）という記号が割り振られ、U+0000 ～ U+10FFFF のような 16 進数による番号によって表される。単純に100万個以上の番号が用意されている。(現状約 13 万個に世界中の文字が割り振り済み)
ex) あ -> UnicodeU+3042
このコードポイントをコンピュータに理解される2進数(16進数)にしたものがUTF
(Unicode は符号化文字集合といい、UTF-8 などは文字符号化方式と呼びます。)


**str型 --string(繋がり/ひと続きの意の単語)
文字列は特定の順序で並ぶ配列およびリストデータ構造と定義されます。
文字列は、文字列は、文字を連続するメモリに格納しているので、[](インデックス演算子)によって各文字にアクセスできる。(0-origin)
ex)"masanao"[2] #=> s
**文字列型のオペランドに対しても演算子は使える e.g.) + (concatenation)を連結演算子として使うことができる。
**固定小数点型 --整数も固定小数点型と言える。


*10進数の指数表記 --3.4 × 10**38 通り は3.4E+38 や 3.4e+38というようにE表記を使っても書ける。
ex)1.67 × 10**-27kg --> 1.67e-27/1.67E-27
**浮動小数点型 --float/double型などがある。

*float型 --単精度浮動小数点型.32ビットで表され、10 進数にして整数部と小数部合わせて約7桁分の値を保持

表現できる値のおおよその範囲は、
-3.4 × 10**38 から -1.2 × 10**-38
0
1.2 × 10**-38 から 3.4 × 10**38


*double型は --倍精度浮動小数点数型とも呼ばれ、64ビットで表され、10 進数にして整数部と小数部合わせて約 15 桁分の値を保持

表現できる値のおおよその範囲は、
-1.8 × 10**308 から -2.2 × 10**-308
0
2.2 × 10**-308 から 1.8 × 10**308
//わずか 64 ビットで、10308 のような非常に大きい数を表現できる。ただし、表現できるビットパターンは 264 個なので、数値を正確に表現できているということではない。


*null --あるデータ型のデータがないこと。物理的にメモリ内にあるデータとは違って、データが存在していないことを null というex)100人アンケートがあったとして、30人が回答します。20人が回答しません。と答えました。この残り50人のまとまりをnilと捉える。

NULLの活用法
例）エラーがないことを示すエラーのデータが存在すれば処理を停止し、エラーのデータが存在しない、つまり null であれば処理を実行する
例）データベースのデータがないことを示す


*void--データ型そのものがない状況。

なにもデータを出力しないプログラムというのも存在します。そもそも何も返さないので、データ型を指定することはできず、「何も出力しない」ということを事前に宣言するというような状況下で使う。

*void/null --null はデータを読み取る際に使われる一方、void はデータを読み取る前に使われ、そのことをプログラムに通知する際に使われます。

***{プリミティブ型(primitive data type)}--------
--bool/char/str/intなどのような、基本的でほとんどのプログラミング言語で提供(組み込まれている)データ型の総称

**リテラル --ソースコードに直接書き込んである文字や数字のこと。コンピュータは、データ(型)の判別を行う必要があるため、通常リテラルには特定の書式を要する(ex)文字列->'')


*文字リテラル --文字の前後をシングルクオテーション（''）で囲む

*文字列リテラル --文字列の前後をダブルクオテーション（""）で囲む

*論理値リテラル --true/falseで表す。PythonではTrue/False


*不動小数点型の誤差 ---0.1 + 0.2というような計算でも、コンピュータで扱うために2進数になおす過程で、循環小数となり、リソースは有限なので、丸め操作をする必要がある。それによって誤差が生じる可能性が常にある。

***{演算子とオペランド}---------
*オペランド(operand) --演算子の対象となるデータ値
*演算子/オペレータ(operator) --予め定義された処理をデータ値に適用し、値を返す記号//文字列のデータ型によって演算子の機能が変わることを演算子のoverloadという。
ex) 3 > 2 #=> true
ex) +-*%は算術演算子
*式(expression) --プログラミング言語において、定められたルールによって、評価される数値と演算子等の組み合わせ。式は評価された値をデータ値として持つ。
ex)8 + 3 //=>11 式は評価された値即ちここでは11を持っている。

*算術計算を行う際に,オペランドのデータ型が異なる場合 --ほとんどの言語では、精度の高い方へ自動的に型変換が行われる
ex)2.0 * 5 //=># 10.0
//int型の5がfloat型のデータ5.0へと変換され、加算 (+) 演算子が適用される。
*殊、除算においては気を付ける
ex)
2 / 5
# Java, C++ : 0 
# JavaScript, PHP, Python : 0.4
 
2.0 / 5
# 0.4
 
2 / 5.0
# 0.4
 
2.0 / 5.0
# 0.4


**優先順位(precedence) --演算子が実行される順序のこと(言語によって定められている)
**結合法則(associativity) --式中に同じ優先順位の演算子が存在する場合、左右どちらから評価を行うかルール
