# Sedコマンドとは?

Stream EDitorの略

`sed`コマンドは、"Stream EDitor"の略で、テキストファイルを編集するためのコマンドラインユーティリティです。主に、テキストファイル内の文字列の検索、置換、削除などの操作を行うために使用されます。

基本的な`sed`コマンドの構文は以下のようになります:

```Shell
sed [オプション] 'コマンド' ファイル名
```

よく使われるオプションには以下のようなものがあります:

- `i`: ファイルを直接編集します（インプレース編集）。

- `e`: 複数のコマンドを連続して実行します。

`sed`コマンドでは、各コマンドを単一引用符（`'`）で囲みます。コマンドは、アドレス、オペレーター、および引数で構成されます。

例えば、`s`オペレーターは文字列の置換に使用されます。構文は以下のようになります:

```Shell
sed 's/検索文字列/置換文字列/g' ファイル名
```

- `s`: 置換オペレーターを表します。

- `/`: 区切り文字として使用されます。他の文字（例えば、`#`や`|`など）も使用可能です。

- `g`: グローバルフラグで、ファイル内の全ての一致する文字列を置換します。

具体的な例として、`pubspec.yaml`ファイル内のバージョン番号を更新する`sed`コマンドを見てみましょう:

```Shell
sed -i -e "s/^version:.*/version: ${{ github.event.inputs.version }}/" pubspec.yaml
```

### このコマンドの詳細:

- `i`: ファイルを直接編集します。

- `e`: 複数のコマンドを連続して実行します（ここでは1つのコマンドのみ）。

- `s`: 置換オペレーターを表します。

- `^version:.*`: 正規表現パターンで、行頭の`version:`に続く任意の文字列にマッチします。

- `version: ${{ github.event.inputs.version }}`: 置換後の文字列で、`${{ github.event.inputs.version }}`はGitHub Actionsの入力変数の値に置き換えられます。

このコマンドを実行すると、`pubspec.yaml`ファイル内の`version:`で始まる行が、指定された新しいバージョン番号に置き換えられます。

`sed`コマンドは非常に強力で、様々なテキスト処理タスクに使用できます。正規表現のサポートにより、柔軟なパターンマッチングと置換が可能です。
