# メール統合移行プラン

> **現状**: Gmail（真面目系メイン） + Outlook（趣味系メイン、一部混在）
> **方針**: Gmail 2本体制（真面目用 + 趣味用）で責務を分離し、Outlook から脱却
> **クライアント**: Spark Mail を導入予定（事務返信の効率化。移行とは独立して進める）
> **SimpleLogin**: 当面なし。必要を感じたら後から導入

---

## 設計: メールアドレスの責務分離

### 構成

```text
serious@gmail.com（既存）  → 真面目系: Dev, 金融, 重要な契約
casual@gmail.com（新規）   → 趣味系: SNS, 買い物, サブスク, ニュースレター
```

Spark でマルチアカウント統合し、操作感は1つの受信箱にする。

### 分類ルール

| カテゴリ          | 使うアドレス | 例                                        |
| ----------------- | ------------ | ----------------------------------------- |
| **真面目-Dev**    | serious      | GitHub, AWS, GCP, Docker Hub, npm, Vercel |
| **真面目-金融**   | serious      | 銀行, クレカ, 証券, 保険, 税務            |
| **真面目-その他** | serious      | ドメイン管理, 確定申告, 重要な契約        |
| **趣味-SNS**      | casual       | Twitter/X, Instagram, Discord             |
| **趣味-買い物**   | casual       | Amazon, 楽天, メルカリ                    |
| **趣味-サブスク** | casual       | Netflix, Spotify, ゲーム系                |
| **趣味-その他**   | casual       | ニュースレター, コミュニティ              |

### 境界が曖昧なサービスの判断基準

迷ったら「**クレカ情報や本人確認書類を預けているか？**」で判断する。

| サービス              | 判断                                           |
| --------------------- | ---------------------------------------------- |
| Apple ID              | serious（決済・端末管理の中核）                |
| Google アカウント自体 | serious（既存のメインアカウント）              |
| Amazon                | casual（クレカ紐づきだが漏洩時に変更しやすい） |
| PayPal                | serious（金融サービス）                        |
| Uber / Grab           | casual（決済は紐づくが本人確認書類はない）     |

> **原則**: 判断に迷ったら casual にしておく。casual → serious への変更は簡単だが、
> serious に趣味系を混ぜると後から分離しにくい。

## Phase 1: Outlook → Gmail 転送設定（Day 2）

移行期間中のセーフティネット。

- [ ] Outlook Web (outlook.com) にログイン
- [ ] 設定 → メール → 転送 → **既存 Gmail（serious）** を転送先に設定
- [ ] 「転送されたメッセージのコピーを保持する」を **ON** にする
- [ ] テストメールを Outlook アドレス宛に送り、Gmail に届くか確認
- [ ] 転送が動いているか週次で確認する（Microsoft が予告なく止めることがある）

> **転送先は serious にする**: 趣味系メールも一旦 serious に届くが、
> Phase 2 でサービス側のメアドを casual に変更すれば自然と分かれていく。
> 金融系が Outlook に残っている可能性があるため、serious で受けるのが安全。

## Phase 2: サービス移行（Day 3〜14）

1日3〜5件ペースで無理なく進める。金融系は待ち時間が発生するので早めに着手。

### 2-1. 優先度 HIGH — 金融・セキュリティ系

Outlook に登録していた重要サービスを **serious** に変更する。

- [ ] クレジットカードの通知先 → serious
- [ ] 銀行・証券口座の通知先 → serious
- [ ] 保険・税務系 → serious
- [ ] 2要素認証のバックアップメール → serious
- [ ] Apple ID（もし Outlook だった場合）→ serious
- [ ] PayPal 等の決済サービス → serious

> **注意**: 金融系はメアド変更に本人確認が必要なことが多い。
> 日本の金融機関は郵送確認で数日かかる場合がある。
> 変更後、確認メールが serious に届くことを必ず検証する。

### 2-2. 優先度 MEDIUM — Dev系の確認

Gmail（serious）に既に登録されているはずだが、念のため確認。

- [ ] GitHub のメール設定を確認（通知先、コミットメール）
- [ ] AWS / GCP のアカウントメール確認
- [ ] npm / Docker Hub / Vercel 等
- [ ] ドメインレジストラ（Cloudflare, お名前.com 等）

### 2-3. 優先度 LOW — 趣味系を casual に移行

Outlook で登録していた趣味サービスを **casual** に変更する。

- [ ] Amazon
- [ ] Twitter/X
- [ ] Netflix / Spotify / YouTube Premium 等
- [ ] Discord
- [ ] ゲーム系（Steam, PlayStation, Nintendo 等）
- [ ] ショッピング系（楽天, メルカリ等）
- [ ] ニュースレター購読

> **Tips**: メアド変更のタイミングでパスワードも更新し、
> パスワードマネージャーに集約するとよい。

### 2-4. リカバリーメールの確認

- [ ] 各サービスのリカバリーメール / バックアップメールに Outlook が設定されていないか確認
- [ ] 設定されていたら serious または casual に変更

## Phase 3: 検証 & クリーンアップ（Day 15〜）

- [ ] Outlook の受信トレイを毎日チェック（まだ直接届くメールがないか）

- [ ] Outlook アカウント自体は **削除しない**（過去のメール参照用に残す）

## 移行後の運用ルール

### 新しいサービスに登録するとき

```text
クレカ・本人確認が関わるサービス → serious
それ以外                        → casual
迷ったら                        → casual（後から serious に変更は簡単）
```

### トラブル対応

| 状況                        | 対応                                                   |
| --------------------------- | ------------------------------------------------------ |
| casual で迷惑メールが増えた | Gmail フィルタでブロック / 配信停止。serious は無傷    |
| 情報漏洩通知が来た          | 該当サービスのパスワード即変更 / 必要ならメアドも変更  |
| casual のアドレスが流出     | 最悪の場合 casual を捨てて新規作成。serious は影響なし |
