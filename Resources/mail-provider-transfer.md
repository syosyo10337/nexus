> **現状**: Gmail（真面目系メイン） + Outlook（趣味系メイン、一部混在）
> **目標**: Gmail 1本 + SimpleLogin（趣味系エイリアス） + Spark クライアント
> **期間**: 1〜2週間（4フェーズ）

## Phase 1: インフラ構築（Day 2〜3）

### 1-1. Outlook → Gmail 転送設定

すべての Outlook 宛メールを Gmail に転送する。これにより移行期間中もメールを見逃さない。

- [ ] Outlook Web (outlook.com) にログイン
- [ ] 設定 → メール → 転送 → Gmail アドレスを設定
- [ ] 「転送されたメッセージのコピーを保持する」を **ON** にする
- [ ] テストメールを Outlook アドレス宛に送り、Gmail に届くか確認

> **重要**: この転送は移行完了まで維持する。最低1ヶ月は切らない。

### 1-2. SimpleLogin セットアップ

- [ ] <https://simplelogin.io> でアカウント作成（無料プラン = 10エイリアス）
- [ ] 転送先メールアドレスとして Gmail を登録・認証
- [ ] 趣味系サービス用のエイリアスを作成（命名規則を決める）

**エイリアス命名規則の推奨:**

```text
[サービス名].[ランダム]@simplelogin.com

例:
  amazon.k7x2@simplelogin.com
  twitter.m9p1@simplelogin.com
  netflix.q3w8@simplelogin.com
```

> サービス名を入れておくと、どこから来たメールか一目でわかる。
> ランダム部分はSimpleLoginが自動生成してくれるのでそれでOK。

- [ ] Gmail 側でフィルタを作成:
  - 条件: `to:*@simplelogin.com`
  - アクション: ラベル「趣味」を付与、受信トレイをスキップ（任意）

### 1-3. Spark Mail インストール

- [ ] Mac: <https://sparkmailapp.com> からダウンロード
- [ ] iOS: App Store からインストール
- [ ] Gmail アカウントを接続
- [ ] Smart Inbox の動作を確認（Personal / Notifications / Newsletters の自動分類）
- [ ] （任意）Outlook アカウントも一時的に Spark に追加
      → 移行期間中は両方のアカウントを Spark で一元閲覧できて便利

## Phase 2: サービス移行（Day 4〜10）

**1日3〜5件ペースで無理なく進める。**

### 2-1. 優先度 HIGH — 金融・セキュリティ系を先に移行

Outlook に登録していた重要サービスのメアドを Gmail に変更する。

- [ ] クレジットカードの通知先 → Gmail に変更
- [ ] 銀行・証券口座の通知先 → Gmail に変更
- [ ] 二要素認証のバックアップメール → Gmail に変更
- [ ] Apple ID（もし Outlook だった場合）→ Gmail に変更

> **注意**: 金融系はメアド変更に本人確認が必要なことが多い。
> 変更後、確認メールが Gmail に届くことを必ず検証する。

### 2-2. 優先度 MEDIUM — Dev系の確認

Gmail に既に登録されているはずだが、念のため確認。

- [ ] GitHub のメール設定を確認（通知先、コミットメール）
- [ ] AWS / GCP のアカウントメール確認
- [ ] npm / Docker Hub / Vercel 等
- [ ] ドメインレジストラ（Cloudflare, お名前.com 等）

### 2-3. 優先度 LOW — 趣味系を SimpleLogin エイリアスに移行

Outlook で登録していた趣味サービスを SimpleLogin 経由に切り替え。

- [ ] 各サービスのアカウント設定 → メアドを SimpleLogin エイリアスに変更
- [ ] Amazon
- [ ] Twitter/X
- [ ] Netflix / Spotify / YouTube Premium 等
- [ ] Discord
- [ ] ゲーム系（Steam, PlayStation, Nintendo 等）
- [ ] ショッピング系（楽天, メルカリ等）
- [ ] ニュースレター購読

> **Tips**: 無料プランは10エイリアスまで。足りなくなったら：
>
> - 重要度の低いサービスは1つのエイリアスを共有してOK
>   （例: `shopping.xx@simplelogin.com` を複数の通販サイトで使う）
> - Premium（$30/年）にすればエイリアス無制限

## Phase 3: 検証 & クリーンアップ（Day 11〜14）

### 3-1. 取りこぼし確認

- [ ] Outlook の受信トレイを毎日チェック（転送されたメールが届いているか）
- [ ] 「このサービスからまだ Outlook にメールが来てる」→ 都度メアド変更
- [ ] 1週間経っても新しいサービスが見つからなくなったら次へ

### 3-2. Spark の最適化

- [ ] Smart Inbox の分類精度を確認。誤分類があれば手動で修正（学習される）
- [ ] 通知設定の調整:
  - 真面目系: 通知 ON
  - 趣味系: 通知 OFF または「まとめて通知」
- [ ] 不要なメルマガの解除（Spark の一括解除機能 or Unsubscribe リンク）

### 3-3. Outlook の段階的フェードアウト

- [ ] Spark から Outlook アカウントを削除（転送があるので Gmail で受信継続）
- [ ] Outlook の転送設定は **最低1ヶ月は維持**
- [ ] 1ヶ月後、Outlook に直接届くメールがほぼゼロになったら転送を停止
- [ ] Outlook アカウント自体は **削除しない**（過去のメール参照用に残す）

## 移行後の運用ルール

### 新しいサービスに登録するとき

```text
真面目なサービス → Gmail アドレスを直接使う
趣味・買い物系   → SimpleLogin でエイリアスを作って登録
判断に迷ったら   → SimpleLogin 経由（後からGmail直に変更は簡単）
```

### トラブル対応

| 状況                         | 対応                                                      |
| ---------------------------- | --------------------------------------------------------- |
| 迷惑メールが増えた           | SimpleLogin でそのエイリアスを無効化                      |
| 情報漏洩通知が来た           | どのエイリアスが漏れたか特定 → 無効化 → 新エイリアス発行  |
| SimpleLogin が使えなくなった | Gmail に直接切り替え（転送先が Gmail なので影響は限定的） |
