# 非同期関数asyncキーワード

asyncが追加関数は非同期関数であり、戻り値はFuture<T>である。

また、このときasync関数内でawaitを実行すると、非同期関数の実行を一時停止して、Futureが完了するまで待機するよ。

```Dart
Future<void> A() async {
  return Future.delayed(Duration(seconds: 1), () {
    print('A');
  });
}

Future<void> B() async {
  return Future.delayed(Duration(seconds: 1), () {
    print('B');
  });
}

Future<void> C() async {
  return Future.delayed(Duration(seconds: 1), () {
    print('C');
  });
}

void main() async {
  await A();
  await B();
  await C();
}
#=>
A
B
C
```

```Dart

main() async {
  await A();
  B();
  await C();
}


#=>
A
C
B
もしくは
A
B
C
Bの完了を待つことなく、Cを実行するためB/Cの順序は保証されない。
```

# Future.microTask

優先的に非同期しょりさせるためらしい。

`Future.microtask`は、Dartの非同期プログラミングにおいて、イベントループの現在のイテレーションが完了した直後に実行されるタスクを登録するための静的メソッドです。

主な特徴は以下の通りです：

1. `Future.microtask`に渡された関数は、現在のイベントループのイテレーションが完了した直後、かつ次のイベントループのイテレーションが開始される前に実行されます。

2. マイクロタスクは、通常のイベントループのタスクよりも優先度が高く、イベントループの各イテレーションの終了時に実行されます。

3. マイクロタスクは、キューに追加された順序で実行されます。

4. マイクロタスクは、UIイベントの処理やアニメーションフレームの更新など、他の重要なタスクを遅延させずに即座に実行したい場合に使用されます。

例えば、以下のようなコードがあるとします：

```Dart
void main() {
  print('Start');
  Future.microtask(() => print('Microtask 1'));
  Future(() => print('Future 1'));
  Future.microtask(() => print('Microtask 2'));
  Future(() => print('Future 2'));
  print('End');
}
```

出力結果は以下のようになります：

```Plain
Start
End
Microtask 1
Microtask 2
Future 1
Future 2
```

これは、`Future.microtask`に渡された関数が、通常の`Future`よりも先に実行されることを示しています。

ただし、マイクロタスクを過度に使用すると、イベントループがブロックされ、アプリケーションのパフォーマンスが低下する可能性があるので注意が必要です。マイクロタスクは、本当に即時に実行する必要があるタスクにのみ使用するべきです。
