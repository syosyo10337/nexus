# ポインタ周りの理解(HEADとか)

# HEAD

現在作業している場所を示すポインタ(参照)。実態はファイルで、  
HEADファイルは、現在作業中のブランチに対するシンボリックリンクです。(ブランチ自身も最新コミットを指すラベルなので、より厳密には)カレントブランチの最新コミットを指すラベル。

```Plain
e.g.)
cat .git/HEAD
ref: refs/heads/main
```

# ^ / ~ の意味

### ~（チルダ）

は親となるコミットを参照できる。

e.g.)

```Bash
HEAD~(==HEAD~1)　# HEADの一個前のコミット(親)
HEAD~~(==HEAD~2)
```

### ^(キャレット)

ブランチのマージにより複数同世代の親が存在する時に指定する

## detached HEAD

git checkoutで、HEAD~1などにあたる過去のコミットに行くと、そこがHEADになり、その状態をdetached HEADと呼ぶ
