# Lookaround

肯定の先読みとかあと読みとか、否定の後読み先読みについて総称として”lookaround”と呼ぶそうな。

どのような機能かというと、文字列をマッチングするときに、前後の文字列を含めることなく、ある文字列の前後に何かしらのパターンがあるかをチェックする機能です。

具体的には、lookaheadとlookbehindに分かれます。

- lookahead: 指定したパターンの後にある文字列を検索する。

- lookbehind: 指定したパターンの前にある文字列を検索する。

lookaroundは、マッチングした部分文字列を含めずに前後の文字列だけをチェックするので、正規表現内で使われる文字列を捕捉することはありません。

## Positive lookahead(先読み)

構文は: `X(?=Y)`

`X` を探し、その直後に`Y`があるかをチェックする。

- ある場合には、マッチするのは`X`だけ

- ない場合にはマッチはスキップされ次の検索に続く。

```JavaScript
e.g.)
\d+(?=\s)(?=.*30)

//1つ以上の数字の直後にスペースが存在して、かつ
//それ以降の何処かに30がある\dを探す。
let str = "1 turkey costs 30€";

alert( str.match(/\d+(?=\s)(?=.*30)/) ); // 1
```

## Negative lookahead(否定の先読み)

構文は `X(?!Y)`

`X`を探すが、`Y` が続かない場合のみ" マッチさせる。

```JavaScript
let str = "2 turkeys cost 60€";

alert( str.match(/\d+(?!€)/) ); 
// 2 (価格部分はスキップされました)
```

# Lookbehind

lookaheadはaheadに対して、何が続くのか？という条件付けをする正規表現です。

lookbehindは、後ろを対象としながら、その前に何が続くのかという条件付けをするための正規表現です。

### Positive lookbehind

`(?<=Y)X` は `X` の前に `Y` がある場合にのみマッチすることを意味します。

### Negative lookbehind

`(?<!Y)X` は `X` の前に `Y` がない場合にのみマッチすることを意味します。
