## *nilガード(自己代入) `|| =`

`A || A = x`の省略形。これも式評価が終わるタイミングを利用した記述。

Aが`nil`もしくは`false`の時は、xを代入。Aが`true`の時はその時点で評価が終わり、Aそのものを返す。

クラス内部で使うような際には、メモ化や遅延初期化を目的をすることが多い。

- メモ化 (データの保持)

```Ruby
def twitter_data 
 @twitter_data || = begin
 # Twitter APIからデータを取得する処理
 #
 #
end
```

ここでは、@twitter_dataに値が存在しない時は、|| =以降の処理を行うが、そうではないときは、インスタンス変数に保持することで、複数回呼び出される時には、ただ、変数に格納された値を返すようになる。←パフォーマンスが向上する。

---

- 遅延初期化

@変数と||=を使って、initializeメソッド内で重い処理を実行されることを避けるテクニック

```Ruby
#遅延初期化を使わない時

class Foo
	attr_reader :bar

	def initialize
		@bar = #何かしらの重い処理を伴って@barを初期化
	end
end
```

```Ruby
#遅延初期化を使う時

class Foo
	#initalizeメソッド内で、@barを初期化しない
	def initialize
	end
	
	def bar
		@bar || = #何かしらの重い処理を伴って@barを初期化
	end
end
```

こうすることで、barメソッドが実行されるタイミングで初めて、@barを初期化を行うため、Foo.newの度毎回重い処理をすることがなくなる。
