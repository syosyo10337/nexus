# 重複実行をやめる

## Concurrencyによる制御

`concurrency`を使用して、同じワークフローの重複実行を防ぐこ
とができます。

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.head_ref || github.ref }}
  cancel-in-progress: true
```

## Push/PRの両方で重複実行を避ける方法

```yaml
on:
  push:
    branches:
      - master
  pull_request:
    branches:
      - master
```
