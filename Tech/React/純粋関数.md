論理的な定義としてはこうなっているらしい。

- fn(決まった引数) → 決まった戻り値

- 関数外の状態(data)を参照変更しない。

- 関数外に影響を及ぼさない。

- "引数で渡された値を変更しない。(Immutabilityの保持)"
    
    - だから、関数内で別途変数を定義し直して、そこに格納すると良いらしい。

    e.g.)

    const newObj = { ...oldObj};
    
    
    上記の要件を満たさない操作を"副作用(side effect)"と呼ぶ。

以上の条件を満たさない操作のことを副作用。

## immutableな値

文字列、数値、BigInt,真偽値、 undefined, Symbol

immutableな場合には、新しい値に対して、メモリのスペースが確保されて、変数に対しての参照が書き換える。

## mutable

書き換えが可能な値

Objectとか配列

メモリ空間に対しては、それぞれの要素(immutableな)分のメモリ空間が確保された上で、その上で、

その組み合わせを表すメモリ空間が存在する。変数自体はその組み合わせを保持しているメモリ空間への参照を保持しているため。
