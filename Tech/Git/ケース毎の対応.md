[復元する方法]  
$ git restore [ファイル名] #commit前の変更を戻す

$ git restore -S [ファイル名]#staging前の状態に戻す。(-S/--staged)  
$ git restore --staged [ファイル名]

$ git checkout HEAD .  
#HEAD(最新コミットの状態)に戻す。エディタの誤操作で作業ディレクトリでファイルを削除したときなど

?*git reset HEAD --ギットの管理下に再度おく。ただこの状態だとローカル上のファイルは削除されたまま。なので、-->  
?*git checkout [ファイル名] --ローカル上にもファイルを戻しておく。  
**$ git checkout -f --ステージングに追加されていない変更を強制的に元に戻す。ex)誤ってファイル消去とかしちゃってステージングもコミットもしてない時

?$ git restore -s HEAD^ foo.txt

???**管理しないファイルをgitの管理から外すには？ --.gitignoreファイルに指定すること。(管理したくないファイルの例:自動生成されるファイル/パスワードがのってるファイル(githubに上がるぞ？))  
--(how to: # [ファイル名] --指定したファイルを除外指定or /[ディレクトリ]　--ルートディレクトリを除外指定or #[ディレクトリ名]/ --ディレクトリ以下を除外指定

- *誤ってコミットしたファイルを管理から外す  
    ①ファイルごとイク場合  
    -*git rm　[ファイル名]をして、ファイルが削除されたよということをステージングエリアに伝えて管理から外れるようにする。(ローカルからファイルを消しつつ、git上でも削除（管理から外すようにする）)  
    -*git rm -r　[ディレクトリ名]をして、ディレクトリが削除されたよということをステージングエリアに伝えて管理から外れるようにする。(ローカルからディレクトリを消しつつ、git上でも削除（管理から外すようにする）)  
    ②ローカルのファイルは残した状態で、  
    *-git rm --cached [ファイル名]を実行して、gitに管理下から一度外すように伝える。但し、.gitignoreに追記しない限り一生無視されるわけじゃない。
