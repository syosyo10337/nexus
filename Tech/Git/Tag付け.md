# タグ

---

コミットをSHA1のハッシュ値ではなく、名前付けすることで区別しやすくするためのもの。文字通りタグ。  
タグには２種類存在し、

- 軽量タグ
    
    - 名前がつけられる
    

- 注釈付きタグ
    
    - 名前を付けられる
    
    - コメントを付けられる
    
    - 署名を付けられる
    

- 使い分けとしては、軽量タグはローカルでの使い捨てなどを行う時用。リリースタグ(ver1.2.3)については注釈付きタグを使うらしい。

### タグの一覧表示

```Bash
# タグ一覧を表示
$ git tag

# -n オプションでタグメッセージも表示
# tag_messageがあるときは、そちらをないときはコミットメッセージ?
$ git tag -n

# ログをタグ付きで確認する(オプションつけなくても、表示される?)
$ git log --decorate
```

### 1. 軽量タグ付け

```Bash
# HEADが指すコミットに対して、タグをつける
$ git tag <tag_name>
```

### 2. 注釈タグ付け

```Bash
# -a: --annotate エディタが起動し、タグにメッセージを埋め込める
# -m: --message コマンドライン上でメッセージを付与できる
$ git tag -a <tag_name>

e.g.)
$ git tag -am "tag_message" tag_name
```

### タグの削除

```Bash
$ git tag -d <tag_name>
```
