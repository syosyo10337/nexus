# テスト戦略

Private関数をテストしないことについて

- **Public APIのみテスト** → リファクタリング耐性が高い

- **実際の使用方法に近い** → バグを見つけやすい

- **振る舞い駆動** → shouldRetryの実装が変わっても、振る舞いが同じならテストが通る

- **モックは最小限** → `queryFn`のモックのみで、TanStack Queryの実際の動作を検証
