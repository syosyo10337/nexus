# ポインタについて

**ポインタ = メモリ上のアドレス（住所）を保持する変数**  

はじめに、Rubyとの比較から始める。

## Rubyとの根本的な違い

### Rubyの場合（全て参照渡し）

```Ruby
class Concert
  attr_accessor :title
  
  def initialize(title)
    @title = title
  end
end

concert = Concert.new("Rock Festival")  # オブジェクトへの参照
another = concert                        # 同じオブジェクトを参照
another.title = "Jazz Festival"
puts concert.title  # => "Jazz Festival" （変更される）
```

### Goの場合（値渡しが基本）

```Go
type Concert struct {
    Title string
}

concert := Concert{Title: "Rock Festival"}  // 値そのもの
another := concert                          // コピーが作られる
another.Title = "Jazz Festival"
fmt.Println(concert.Title)  // => "Rock Festival" （変更されない！）
```

# ポインタとアドレス演算子

## _`&`_ _: アドレス演算子（変数のアドレスを取得）_

**Address-of operator**

## _`*`_ _: 間接参照演算子（ポインタが指す値を取得）_

**Dereference operator** (間接参照演算子)

### 視点

(Ruby)参照渡しとは、ポインタをやり取りすること。

## Rubyエンジニアが押さえるべきポイント

### ✅ Go言語の慣習

```Go
// 1. 構造体のコンストラクタは通常ポインタを返す
func NewSomething() *Something {
    return &Something{}
}

// 2. メソッドレシーバーの選び方
func (s *Something) Mutate() {}    // 状態を変更する場合
func (s Something) ReadOnly() {}    // 読み取り専用の場合// 3. nilチェックが必要
if endpoints != nil {
    // 安全に使用
}
```

### ✅ 言語設計の哲学の違い

**Ruby**: 「開発者の幸せ」を重視

ruby

_`# ポインタを意識させない# すべてがオブジェクト、すべてが参照`_`    array = [1, 2, 3]   hash = {key: "value"}    `_`# 裏では全部ポインタだが、見えない`_

**Go**: 「明示的であること」を重視

go

_`// 値かポインタか明確に区別`_`value := Concert{}` _`// 値（スタック上）`_`pointer := &Concert{}` _`// ポインタ（ヒープ上）`_

## まとめ：実践的な指針

### ✅ Go言語のベストプラクティス

1. **デフォルトは値**
    
    - まず値で考える
    
    - 問題があればポインタ
    

2. **ポインタを使う場合**
    
    - 大きな構造体（目安：64バイト以上）
    
    - 状態を変更する必要がある
    
    - nil を表現したい
    
    - 複数箇所で同一インスタンスを共有
    

3. **メモリは気にしすぎない**
    
    - コンパイラが最適化
    
    - プロファイリングして初めて最適化
    

これらの使い分けができると、効率的でGoらしいコードが書けるようになります！
