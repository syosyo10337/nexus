# データの標準化

---

回帰式を見た時に、

y=1.076456(′SNS1′)+　0.534002(′SNS2′)+0.284738(′actor′)+213.95584(′original′)+6253.418729438708y = 1.076456('SNS1') +　\\ 0.534002('SNS2') + \\ 0.284738('actor') + \\ 213.95584('original') + 6253.418729438708y=1.076456(′SNS1′)+　0.534002(′SNS2′)+0.284738(′actor′)+213.95584(′original′)+6253.418729438708

このような値があるとすれば、’sns1に１つの投稿をすること’ と’originalの作品が存在するか’の双方の間に、変数(parameter)の重さに差がある(オリジナルひとつ作る労力と、sns投稿の労力の差)

専門的な言い方をすると、**各特徴量の分布が大きく異なる**

e.g.)

sns1のavg() 2000 で分布は1900 ~ 2100

originalはavg() 0.51 で分布は　０、１

これは、標準偏差も異なることが予想できる(ばらつき度)

つまり、特徴量ごとに平均値や標準偏差(データのばらつきをそのデータの単位基準で表したもの)が大きく異なると特徴量を比較することができない。

これを解消する作業をデータの標準化と呼ぶ。

具体的には,元データの分布に関わらず

- 平均値0

- 標準偏差が1

の分布にすること。

# 算出法

ある特徴量のデータセットをx1, x2 ,,, 平均値m・標準偏差sd、標準化されたデータセットをz1, z2, ..とする時

z1=x1−msdz1 = \frac{x1 - m}{sd}z1=sdx1−m​

## `sklearn.preprocessing`モジュールでの実行

```Python
e.g.)
from skelearn.preprocessing import StandardScaler

# StandardScalerクラスのオブジェクトを生成する
sc_model_x = StandardScaler()

# fitメソッドにデータフレームを渡すと、
# そのデータフレームの平均値と標準偏差を属性として保持したオブジェクトになる
sc_model_x.fit(x_train)

# transformメソッドにデータセットを渡すとき
# オブジェクトが持つ平均値と標準偏差を元に、標準化されたデータを返す(numpy.array型)。
sc_x = sc_model_x.transform(x_train)
sc_x
```

### 標準化したデータを元に戻す

```Python
# 上の例で見るように、numpy.array型のデータセットだと仮定する
sc_x.inverse_tranform()
```

この後、正解データ(y_train)についても標準化する

sklearnの仕様上 `sc_model.fit()`の引数がdataframeである必要がある

# 標準化データの注意点

標準化データを用いて学習させたモデルの決定係数を算出するときに、素のvalidation データを使用すると、おかしな数字を取る。

これは、**学習には標準化データを使用したものの、validationデータには、標準化をさせていなかったためである。**

予測性能を評価するためには、valデータについても標準化を施す必要がある。

🚨

標準化をさせる際には、==trainingデータ==の平均値と標準偏差を利用する必要があります。

# 標準化されたデータの参照

```Python
#平均値
print(sc_model_x.mean_)

#標準偏差(分散の√)
print(sc_model_x.scale_)

#分散(標準偏差の２乗)
print(sc_model_x.var_)
```
