# 分割代入引数

[https://typescriptbook.jp/reference/functions/destructuring-assignment-parameters](https://typescriptbook.jp/reference/functions/destructuring-assignment-parameters)

```TypeScript
//関数ないでは、xyで扱うが、オブジェクトのa,bプロパティの値を使うよ。
function foo({ a: x, b: y }) {
  console.log(x, y);
}
foo({ a: 1, b: 2 });
```

**分割代入引数の型注釈**

```TypeScript
function foo({ a, b }: { a: number; b: number }) {}
//                   ^^^^^^^^^^^^^^^^^^^^^^^^^^型注釈

function bar([num1]: number[]) {}
```

## 分割代入引数のデフォルト引数[​](https://typescriptbook.jp/reference/functions/destructuring-assignment-parameters#%E5%88%86%E5%89%B2%E4%BB%A3%E5%85%A5%E5%BC%95%E6%95%B0%E3%81%AE%E3%83%87%E3%83%95%E3%82%A9%E3%83%AB%E3%83%88%E5%BC%95%E6%95%B0)

渡されなくてもデフォルト値使うから a?で、その後に型注釈をつける。

```TypeScript
function foo({ a = 0 }: { a?: number | string }) {}
```

## **分割代入引数の全体の既定値**

💡

TS では、型注釈の後に、全体の default 値を記載します。

```TypeScript
function foo({ a, b } = { a: 0, b: 0 }) {
  console.log(a, b);
}
foo();
foo({ a: 1 });

function bar([a, b] = [0, 0]) {
  console.log(a, b);
}
bar();
bar([1]);


 //                ................型注釈の位置
function foo({ a }: { a?: number } = { a: 0 }) {}
//                                 ^^^^^^^^^^既定値の位置
```

### オプショナルオブジェクトを定義する際の記法

オプショナルオブジェクトを分割代入引数で定義する場合、`?`ではなく`= {}`を使うのが推奨されます。

#### 構文の分解

```TypeScript
function parseCsvFile(
  file: File,
  { headerRowIndex = 0 }: CsvParseOptions = {}
): Promise<ParsedCsv>
```

この第二引数は**3 つの要素**で構成されています：

1. **`{ headerRowIndex = 0 }`** - 分割代入 + プロパティのデフォルト値

   - オブジェクトを分割代入して、`headerRowIndex`プロパティを取り出す
   - `headerRowIndex`が存在しない場合は`0`を使用

2. **`: CsvParseOptions`** - 型注釈

   - このパラメータの型は`CsvParseOptions`

3. **`= {}`** - **パラメータ全体**のデフォルト値
   - 引数が渡されなかった場合、空オブジェクト`{}`を使用

#### なぜ `?` を使わないのか

##### ❌ `?` を使った場合（これはエラーになる）

```TypeScript
function parseCsvFile(
  file: File,
  { headerRowIndex = 0 }: CsvParseOptions?  // ⚠️ 構文エラー
): Promise<ParsedCsv>

// または
function parseCsvFile(
  file: File,
  { headerRowIndex = 0 }?: CsvParseOptions  // ❌ ランタイムエラーの可能性
): Promise<ParsedCsv>
```

**問題点：**

```TypeScript
parseCsvFile(file);  // 第二引数が undefined

// 内部で undefined を分割代入しようとする
// const { headerRowIndex = 0 } = undefined;  // ❌ エラー！
// TypeError: Cannot destructure property 'headerRowIndex' of 'undefined'
```

##### ✅ `= {}` を使った場合（正しい）

```TypeScript
function parseCsvFile(
  file: File,
  { headerRowIndex = 0 }: CsvParseOptions = {}
): Promise<ParsedCsv>
```

**動作：**

```TypeScript
// 引数を省略
parseCsvFile(file);
// ↓ 内部的には
// const { headerRowIndex = 0 } = {};  // ✅ OK！
// headerRowIndex = 0 (デフォルト値が使われる)

// 引数を渡す
parseCsvFile(file, { headerRowIndex: 2 });
// ↓ 内部的には
// const { headerRowIndex = 0 } = { headerRowIndex: 2 };
// headerRowIndex = 2

// 空オブジェクトを渡す
parseCsvFile(file, {});
// ↓ 内部的には
// const { headerRowIndex = 0 } = {};
// headerRowIndex = 0 (デフォルト値が使われる)
```

#### 2 段階のデフォルト値

この構文は**2 段階のデフォルト**を設定しています：

**第 1 段階：パラメータ全体のデフォルト（`= {}`）**

```TypeScript
// 引数を省略した場合
parseCsvFile(file)
// ↓
// 第二引数 = {}（空オブジェクトがデフォルト）
```

**第 2 段階：プロパティのデフォルト（`= 0`）**

```TypeScript
// headerRowIndexプロパティがない場合
{ headerRowIndex = 0 }
// ↓
// headerRowIndex = 0（0がデフォルト）
```

#### 実際の動作例

```TypeScript
interface CsvParseOptions {
  headerRowIndex?: number;
  delimiter?: string;
}

function parseCsvFile(
  file: File,
  { headerRowIndex = 0, delimiter = ',' }: CsvParseOptions = {}
): void {
  console.log('headerRowIndex:', headerRowIndex);
  console.log('delimiter:', delimiter);
}

// ケース1: 引数を完全に省略
parseCsvFile(file);
// 出力:
// headerRowIndex: 0
// delimiter: ,

// ケース2: 空オブジェクトを渡す
parseCsvFile(file, {});
// 出力:
// headerRowIndex: 0
// delimiter: ,

// ケース3: 一部のプロパティを指定
parseCsvFile(file, { headerRowIndex: 2 });
// 出力:
// headerRowIndex: 2
// delimiter: ,

// ケース4: すべてのプロパティを指定
parseCsvFile(file, { headerRowIndex: 3, delimiter: ';' });
// 出力:
// headerRowIndex: 3
// delimiter: ;
```

#### 他のパターンとの比較

**パターン A: `= {}` を使う（✅ 推奨）**

```TypeScript
function parse(
  { headerRowIndex = 0 }: Options = {}
) {
  // headerRowIndexは常に数値
}

parse();              // ✅ OK
parse({});            // ✅ OK
parse({ headerRowIndex: 2 }); // ✅ OK
```

**パターン B: `?` を使う（⚠️ エラーになる）**

```TypeScript
function parse(
  { headerRowIndex = 0 }?: Options
) {
  // headerRowIndexは常に数値
}

parse();              // ❌ TypeError: Cannot destructure...
parse(undefined);     // ❌ TypeError: Cannot destructure...
parse({});            // ✅ OK
parse({ headerRowIndex: 2 }); // ✅ OK
```

**パターン C: `?` + オプショナルチェーン（⚠️ 冗長）**

```TypeScript
function parse(
  options?: Options
) {
  const headerRowIndex = options?.headerRowIndex ?? 0;
  // 追加の行が必要
}

parse();              // ✅ OK
parse(undefined);     // ✅ OK
parse({});            // ✅ OK
parse({ headerRowIndex: 2 }); // ✅ OK
```

#### まとめ

```TypeScript
{ headerRowIndex = 0 }: CsvParseOptions = {}
```

この構文は：

1. **`= {}` でパラメータ全体をオプショナル化**
   - `?`の代わりに、デフォルト値`{}`を提供
   - 引数省略時に`undefined`ではなく`{}`が使われる
2. **分割代入が安全に動作**

   - `{}`を分割代入できる（エラーにならない）
   - `undefined`を分割代入するとエラーになる

3. **プロパティごとのデフォルト値も設定可能**
   - `headerRowIndex = 0`でプロパティのデフォルト値

**つまり、`?`でオプショナルにする代わりに、`= {}`でデフォルト値を提供することで、より安全で簡潔な構文を実現しています。**

**呼び出し時のプロパティ名の省略**

API 通信などで型名を一致させるような際にはこれが使えそうだね。

```TypeScript

function bmi({ height, weight }: { height: number; weight: number }) {}
 
// プロパティ名と同じ変数
const height = 170;
const weight = 65;
 
// プロパティ名を省略しない呼び出し
bmi({ height: height, weight: weight });
 
// プロパティ名を省略した呼び出し
bmi({ weight, height });
```

渡しているのはあくまで props オブジェクトなので、呼び出し側の、プロップスの渡し方はオブジェクトリテラルでも良い。

```TypeScript
        <Profile {...profile[0]} />
```
