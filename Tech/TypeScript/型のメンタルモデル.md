# 型のメンタルモデル

type theoryという数学の研究分野が源流のようなもの。

厳密には異なるが、集合論 (set theory)を密接な関わりがあり、

TypeScriptでは意図的に、型を集合として扱っているような理解ができる作りになっている。

[https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes-oop.html#types-as-sets](https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes-oop.html#types-as-sets)

# 型を集合として解釈する 3つの例

## ユニット型 (unit type)

単位的な型

単位集合(unit set)、単集合(singleton set)

TypeScript語でいうと、nullやundefinedは単一の値、null/undefinedを持っていることからユニット型と言えます。

あとは、リテラル型もそうだね。　

# ボトム型　（bottom type)

値を全く持たない型のこと。

型が集合であるとするとき、ボトム型は空集合(empty set)に相当し、空型(empty type)とも呼ばれることがあります。

# トップ型( top type)

ボトム型が値をまったく持たない型なら、それとは逆にすべての値を持つような型も存在しています。そのような型を**トップ型**(top type)と呼びます。

オブジェクト指向言語であれば大抵は型階層のルート位置、つまりトップ位置に存在している型であり、TypeScriptでは`unknown`型がトップ型に相当します。

ボトム型が空集合に相当するなら、トップ型は全体集合に相当すると言えるでしょう。なおTypeScriptでは`{} | null | undefind`という特殊なユニオン型を`unknown`型相当として扱い、相互に割当可能としています。

その他にはany型とかね。

`any`型の特殊性はトップ型としてあらゆる型からの割当が可能だけでなく、`never`型を除くあらゆる型へも割当可能な点です。

```TypeScript
declare const a: any;

const n1: unknown = a;
const n2: {} = a;
const n3: number = a;
const n4: 1 = a;
const n5: never = a;
Type 'any' is not assignable to type 'never'.
```
