# プロセスとスレッド

## プロセスとは

OS が**プログラムを実行するための単位**。プロセスごとに以下が**独立して**割り当てられる。

```
プロセスA                    プロセスB
┌─────────────────┐        ┌─────────────────┐
│ メモリ空間       │        │ メモリ空間       │
│  - コード        │        │  - コード        │
│  - ヒープ        │        │  - ヒープ        │
│  - スタック      │        │  - スタック      │
│ ファイル記述子   │        │ ファイル記述子   │
│ PID: 1234       │        │ PID: 5678       │
└─────────────────┘        └─────────────────┘
   完全に隔離されている（壁がある）
```

## スレッドとは

プロセスの**中にある実行の流れ**。同じプロセス内のスレッドは**メモリを共有する**。

```
プロセスA
┌───────────────────────────────────┐
│ 共有メモリ空間（コード、ヒープ）    │
│                                   │
│  スレッド1    スレッド2    スレッド3 │
│  ┌──────┐   ┌──────┐   ┌──────┐  │
│  │スタック│   │スタック│   │スタック│  │
│  └──────┘   └──────┘   └──────┘  │
│  （スタックだけ各自持つ）           │
└───────────────────────────────────┘
```

## 比較

| | プロセス | スレッド |
|---|---|---|
| **メモリ** | 各自独立 | 同じプロセス内で共有 |
| **生成コスト** | 重い（OS がメモリ空間を新規作成） | 軽い（メモリ空間を使い回す） |
| **通信** | IPC が必要（パイプ、ソケット等） | 共有メモリで直接やりとり |
| **安全性** | 1つ落ちても他に影響しない | 1つ落ちるとプロセス全体が死ぬ |
| **データ競合** | 起きない（隔離されてる） | 起きる（同じメモリを触るから） |

## マルチプロセス vs マルチスレッド

### マルチプロセス

複数のプロセスを立ち上げて並列に処理する方式。

- プロセス同士はメモリが隔離されているので**安全**
- 1つのプロセスがクラッシュしても他に影響しない
- ただしプロセス間通信（IPC）は重い

**使用例:** Nginx のワーカープロセス、Chrome のタブ

### マルチスレッド

1つのプロセス内で複数のスレッドを走らせる方式。

- スレッド生成は軽量で、メモリ共有による高速な通信が可能
- ただし共有メモリの**データ競合（race condition）** に注意が必要
- 1つのスレッドの致命的エラーがプロセス全体を巻き込む

**使用例:** Java のサーバー、ゲームエンジン

### 使い分けの指針

```
安全性が大事（クラッシュ耐性）  → マルチプロセス
性能が大事（軽量・高速通信）   → マルチスレッド
両方使う場合も多い
  例: Chrome = タブごとにプロセス、プロセス内でスレッド複数
```

## JavaScript / Node.js との対応

| 概念 | JS での対応 |
|---|---|
| マルチプロセス | Chrome の各タブ、Node.js の `child_process.fork()` / `cluster` |
| マルチスレッド | `Web Worker`、Node.js の `worker_threads` |
| プロセス間通信 | `postMessage`（タブ間） |
| 共有メモリ | `SharedArrayBuffer`（Worker 間） |

## シングルスレッド・非同期・並列の違い

| 概念 | 意味 | JS での例 |
|---|---|---|
| **同期 (sync)** | 前の処理が終わるまで次に進まない | `fs.readFileSync(...)` |
| **非同期 (async)** | 待たずに次へ進み、終わったら通知を受ける | `fetch().then(...)` / `await fetch()` |
| **並行 (concurrent)** | 複数タスクを切り替えながら進める（スレッドは1本でも可能） | JS のイベントループ |
| **並列 (parallel)** | 複数の処理が物理的に同時に走る | `Web Worker`、マルチスレッド |

> **非同期 ≠ 並列**。JS の `Promise.all()` は非同期だが並列ではない。1本のスレッドで切り替えているだけ（並行）。
