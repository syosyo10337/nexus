# 正規表現 (Regular Expression)

正規表現とは、[文字列](https://ja.wikipedia.org/wiki/%E6%96%87%E5%AD%97%E5%88%97)の集合を一つの文字列で表現する方法の一つである。

## よく使うもの一覧表

| 表現 | 説明 |
| --- | --- |
| `.` | **Any single character (任意の一文字)**``改行を除く任意の1文字にマッチするメタ文字（半角スペースや @ も含む） |
| `\d` or `[0-9]` | **Any digit (1個の半角数字 (0123456789) を表すメタ文字 (文字クラス))** |
| `\w` | **Any word character (英単語を構成する文字)**Ruby/JavaScript では `\w = [a-zA-Z0-9_]` |
| `\s` | **Any white space character (タブ、改行、半角スペースを含む空白)**Ruby の場合 `\s = [ \t\r\n\f]`（`\r`: 復帰文字, `\f`: 改ページ） |
| `\t` | タブ文字を表す（文字クラス） |
| `a*` | **Zero or more of a (直前の文字が0文字以上連続するもの)**（貪欲） |
| `a+` | **One or more of a (直前の文字が1文字以上連続するもの)**（貪欲） |
| `{n}` | **ちょうど n 回繰り返すとき**（量指定子）e.g. `\w{1,}`: 1回以上文字が繰り返す場合 |
| `{n,m}` | **直前の文字が n 回以上、m 回繰り返すとき**（量指定子）※スペースを入れてはいけません。e.g. `\d{2,5}-\d{1,4}-\d{4}`（郵便番号や電話番号の形式など） |
| `a?` | **Zero or one of a (直前の文字が0もしくは1文字あるもの)**`()` でグループ化した文字列 `(a)` が0もしくは1つある時の指定。自動でキャプチャされるため、グループ化のみの場合は `(?: )` を使用する。e.g. `(ABC)?`: 文字列 ABC があるか、ない時e.g. `モハメド[ .]?アリ`: 「モハメド アリ」や「モハメド.アリ」を許容する |
| `[]` | **[] 内の要素のいずれか一文字を指定**`-`（ハイフン）を使用すると範囲指定（例: `[a-z]`）になる。e.g. `[ABC]`: A, B or C のいずれかe.g. `[-)]`: `-` または `)`（`()` は `[]` 外では別の意味を持つ）e.g. `[A-Za-z0-9]`: 半角英数字1文字 |
| `[^abc]` | **Any single character except: a, b, or c (a, b, c 以外の任意の1文字)**`^` 以降で指定した文字以外の文字列にマッチする。最小量指定子（`*?`, `+?`）を使う方法もある |
| `^` | **行頭を表すメタ文字 (アンカー)**e.g. `/^ +/`: 行頭からスペースが1文字以上続く |
| `$` | **行末を表すメタ文字 (アンカー)**e.g. `/^ +$/`: 行頭から行末までスペースが1文字以上続く |
| `()` | **Capture everything enclosed (囲まれたものをキャプチャする)**`\1`, `\2`: キャプチャされた n 番目の文字列を参照（他言語では `$1`, `$2`）e.g. Ruby: `'hello'.match('(.)\1')` → `(.)` で任意の1文字、`\1` でそれと同じ文字が続くことを意味するe.g. PHP: `preg_match()` でキャプチャしたサブパターンは `$matches` の `[1]` から順に格納される |
| `\u3040-\u309f` | **ひらがな** |
| `\u30a0-\u30ff` | **カタカナ** |
| `\u4e00-\u9faf` | **漢字 (CJK 統合漢字)** |

## 文字クラス

文字の集合のこと。

### [] の中でのメタ文字

`()$.+?|{}` は `[]` 内ではただの記号として扱われるが、`\w`, `\d`, `\s`, `\n` や `-`, `^` は特殊な意味を持ったままなので注意が必要。

### `i` 修飾子

`ignore case` の意味で、大文字小文字を区別しない。

e.g. `/\wAa/i` → 何かしらの単語文字の後に `A`（または `a`）が2回続く場合。

## Lookaround (先読み・後読み)

肯定の先読み・後読み、否定の先読み・後読みを総称して "Lookaround" と呼ぶ。

文字列をマッチングするときに、マッチ結果に含めることなく、あるパターンの前後に特定のパターンが存在するかをチェックする機能。

## Lookahead (先読み)

指定したパターンの後にある文字列を条件にする。

### Positive Lookahead (肯定先読み)

構文: `X(?=Y)`
`X` を探し、その直後に `Y` があるかチェックする。

- ある場合: `X` にマッチする。
- ない場合: マッチしない。

```javascript
// e.g.) \d+(?=\s)(?=.*30)
// 1つ以上の数字の直後にスペースがあり、かつそれ以降のどこかに 30 がある数字を探す
let str = "1 turkey costs 30€";
console.log(str.match(/\d+(?=\s)(?=.*30)/)); // "1"
```

### Negative Lookahead (否定先読み)

構文: `X(?!Y)`
`X` を探すが、その直後に `Y` が**続かない**場合のみマッチさせる。

```javascript
let str = "2 turkeys cost 60€";
console.log(str.match(/\d+(?!€)/)); // "2" (価格部分は € が続くのでスキップされる)
```

## Lookbehind (後読み)

指定したパターンの前にある文字列を条件にする。

### Positive Lookbehind (肯定後読み)

構文: `(?<=Y)X`
`X` の前に `Y` がある場合にのみ `X` にマッチする。

### Negative Lookbehind (否定後読み)

構文: `(?<!Y)X`
`X` の前に `Y` がない場合にのみ `X` にマッチする。

## 練習・ドラフト

`(\d+[smh]|(?:\d+ )?[A-Z][a-z]{2} \d+)`

## アンカー

- `^`: 行頭
  - e.g. `/^ +/`: 行頭からスペースが1文字以上続く
- `\b`: 単語の境界 (boundary)
  - e.g. `\bear\b`: 単語として "ear" にマッチする（"search" の "ear" にはマッチしない）

## 演算子

- `|`: OR 条件
  - e.g. `ABC|DEF`: 文字列 "ABC" または "DEF"

## 後方参照

`()` でキャプチャした文字列を `\1` などで参照すること。

- e.g. `<a href="(.+?)">\1</a>`
  - `(.+?)` でマッチした URL 部分と同じテキストがタグの中身にある場合にマッチする。

## エスケープ

- `\`: 特殊な意味を持つ文字（メタ文字）を普通の文字として扱う。
- Ruby や JavaScript では `/` で囲むため、中身の `/` は `\/` とエスケープする必要がある。

## その他メタ文字

- `\D`: 半角数字以外
- `\W`: 英単語構成文字以外（記号や空白など）
- `\S`: 空白文字以外
- `\B`: 単語の境界以外の位置
