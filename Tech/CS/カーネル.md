# カーネル

## 概要

カーネル（Kernel）はOSの中核となるソフトウェアで、ハードウェアとアプリケーションの間を仲介する。
プロセス管理、メモリ管理、デバイス制御、ファイルシステムなどOSの根幹機能を担う。Linux 固有の概念ではなく、すべてのOSに存在するCS全般の概念。

## カーネルの主な役割

| 機能                 | 説明                                               |
| -------------------- | -------------------------------------------------- |
| **プロセス管理**     | プロセスの生成・終了・スケジューリング             |
| **メモリ管理**       | 仮想メモリ、ページングなどで物理メモリを抽象化     |
| **デバイス管理**     | デバイスドライバを通じてハードウェアを制御         |
| **ファイルシステム** | ディスク上のデータへの読み書きインターフェース     |
| **システムコール**   | ユーザ空間のプログラムがカーネル機能を呼び出す窓口 |

## カーネルモードとユーザモード

CPUには実行モードがあり、カーネルは特権モード（カーネルモード）で動作する。

```text
ユーザモード（制限あり）
┌──────────────────────┐
│  アプリケーション      │
│  (ブラウザ, エディタ)  │
└──────────┬───────────┘
           │ システムコール
           ▼
┌──────────────────────┐
│  カーネルモード（特権） │
│  - プロセス管理       │
│  - メモリ管理         │
│  - デバイスドライバ    │
│  - ファイルシステム    │
└──────────────────────┘
           │
           ▼
┌──────────────────────┐
│  ハードウェア          │
│  (CPU, メモリ, HDD)   │
└──────────────────────┘
```

ユーザモードのプログラムがハードウェアにアクセスするには、**システムコール**を介してカーネルに依頼する必要がある。
Linuxでは、カーネルだけがカーネルモードで動作して、デバイスにアクセスできます。プロセスはユーザモードでアクセスするため、デバイスへアクセスできない。

## カーネルの種類

### モノリシックカーネル

OSの主要機能（ファイルシステム、デバイスドライバ、ネットワーク等）をすべてカーネル空間に含める方式。

- **メリット**: カーネル内の機能間通信が高速（関数呼び出しで済む）
- **デメリット**: カーネルが肥大化しやすく、1つのバグがシステム全体に影響する
- **採用例**: Linux, FreeBSD

### 「カーネル」= Linux ではない理由

「カーネル」が Linux と強く結びつくのは、**Linux がカーネルそのものの名前**だから。GNU/Linux において：

- **GNU** = ユーザランドのツール群（bash, coreutils 等）
- **Linux** = カーネル

他のOSではカーネルに固有名がついている（Windows NT カーネル、XNU など）ため、「カーネル」という一般名詞が前面に出ない。

## システムコール

プロセスがカーネルに処理を依頼するための手段
ファイルシステムの操作やプロセス生成のために、システムコールを用いてカーネルを呼び出す。
