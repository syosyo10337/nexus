## シェルとは

シェルは、カーネルが提供する機能を操作する際に、OSと対話的に操作するためのインターフェースです。シェルはOSの特にカーネル部分を包み込んでいることからその名があり、対話機能を提供します。

### 主要なシェル

| シェル | 説明 |
|--------|------|
| `bash` | Bourne Again Shell（最も一般的） |
| `zsh` | Z Shell（macOSのデフォルト） |
| `sh` | Bourne Shell（POSIX互換） |
| `fish` | Friendly Interactive Shell |

---

## プログラムの主要4要素

シェルスクリプトは、以下の4つの要素で構成されます:

- **順次実行**: コマンドを上から順に実行
- **条件分岐**: 条件に応じて処理を分岐 → [条件分岐.md](条件分岐.md)
- **繰り返し**: 同じ処理を繰り返す → [繰り返し処理.md](繰り返し処理.md)
- **サブルーチン**: 処理をまとめて再利用可能にする（関数） → [関数.md](関数.md)

# スクリプトの作成と実行

## 基本的な作成方法

シェルスクリプトはテキストファイルとして作成します（`vi`や`vim`などのエディタを使用）。

### スクリプトの基本構造

```bash
#!/bin/bash
# 1行目: シェバン（shebang）でシェルを指定

# コメント: #で始まる行はコメント

# 2行目以降: 実行するコマンドを記述
echo "Hello, World!"
date
ls -l
```

### シェバン（Shebang）

| シェバン | 説明 |
|---------|------|
| `#!/bin/bash` | Bashで実行 |
| `#!/bin/sh` | POSIX互換シェルで実行 |
| `#!/usr/bin/env bash` | 環境のbashを使用（推奨） |
| `#!/bin/zsh` | Zshで実行 |

> **推奨**: `#!/usr/bin/env bash`を使用すると、システムに依存せずbashを見つけられます。

## 実行権限の付与

```bash
# 所有者に実行権限を付与（推奨）
$ chmod u+x script.sh

# 全員に実行権限を付与
$ chmod +x script.sh

# 数値指定
$ chmod 755 script.sh  # rwxr-xr-x

# 権限を確認
$ ls -l script.sh
-rwxr-xr-x 1 user group 100 Jan 4 10:00 script.sh
```

> **注意**: スクリプトを実行するには、実行権限（`x`）が必要です。

## スクリプトの実行方法

### 方法1: パス指定で実行

```bash
# カレントディレクトリのスクリプトを実行
$ ./script.sh

# 絶対パスで実行
$ /path/to/script.sh

# PATHが通っているディレクトリなら
$ script.sh
```

### 方法2: シェルを明示的に指定

```bash
# Bashで実行
$ bash script.sh

# shで実行
$ sh script.sh

# 実行権限不要
```

### 方法3: sourceコマンド

```bash
# 現在のシェル環境で実行
$ source script.sh

# または短縮形
$ . script.sh
```

> **sourceの特徴**: 変数や関数が現在のシェルに残ります。詳細は後述。

# 基本コマンドと変数 

## echoコマンド

引数で与えた文字列を標準出力に出力します。

```bash
echo [オプション] 文字列
```

### 主要オプション

- `-n`: 改行を抑制（通常は自動で改行されます）
- `-e`: エスケープシーケンスを有効化

### 使用例

```bash
$ echo "Hello, World!"
Hello, World!

$ echo -n "改行なし"
改行なし$
```

## 変数の基本操作

> **詳細**: シェル変数と環境変数の詳細は[シェル変数.md](シェル変数.md)を参照してください。

### 変数の代入と参照

```bash
# 代入は「=」、参照は「$」を使用
$ abc=123
$ echo $abc
123
```

> **注意**: `=`の前後にスペースを入れないこと（コマンドとして解釈されてしまいます）

### 配列変数

Bashでは1次元の配列変数が使用できます。

```bash
# 配列への代入
$ abc[0]=123
$ abc[1]=456

# 配列の参照（波括弧で囲む）
$ echo ${abc[0]}
123
$ echo ${abc[1]}
456
```

## readコマンド

標準入力からデータを読み込み、変数に格納します。既存のデータは上書きされます。

```bash
read 変数名
```

> **注意**: 変数名の前に`$`は不要です（参照ではなく代入のため）

### 使用例

```bash
$ echo $abc
123

$ read abc
aaabbbccc

$ echo $abc
aaabbbccc
```

## 引用符（クォート）の種類と使い分け

### シングルクォート（`'`）

囲んだ範囲を**文字列として**そのまま認識します。

```bash
$ echo 'Value is $abc'
Value is $abc
```

### ダブルクォート（`"`）

囲んだ範囲の`$`付き変数を**変数として展開**します。

```bash
$ abc=123
$ echo "Value is $abc"
Value is 123
```

### バッククォート（`` ` ``）

囲まれた範囲を**コマンドとして実行**し、その結果を返します。

```bash
$ XYZ=`date`
$ echo "It is $XYZ now."
It is Thu Mar 20 06:08:14 JST 2017 now.

# 変数への代入を省略する書き方
$ echo "It is `date` now."
It is Thu Mar 20 06:08:14 JST 2017 now.
```

> **現代的な書き方**: バッククォートの代わりに`$(command)`の使用が推奨されます。

# スクリプトの引数処理

## 位置パラメータ（引数）

スクリプト実行時に渡された引数を参照できます。

### 特殊変数

| 変数 | 説明 |
|------|------|
| `$0` | 実行コマンド名（スクリプト名） |
| `$1`, `$2`, `$3`... | 1番目、2番目、3番目...の引数 |
| `$#` | 引数の個数 |
| `$@` | すべての引数（個別に展開） |
| `$*` | すべての引数（1つの文字列として展開） |

### 使用例

```bash
#!/bin/bash
echo "$1:" $1
echo "$2:" $2
echo "$3:" $3
echo "$0:" $0
echo "$#:" $#
```

実行結果:

```bash
$ ./args.sh aaa bbb ccc
$1: aaa
$2: bbb
$3: ccc
$0: ./args.sh
$#: 3
```

## shiftコマンド

引数の順序を左にずらします（`$1`を削除し、`$2`が`$1`になる）。

### 使用例

```bash
#!/bin/bash
echo "$1:" $1
echo "$2:" $2
echo "$3:" $3
shift
echo "$1:" $1
echo "$2:" $2
```

実行結果:

```bash
$ ./argsshift.sh aaa bbb ccc
$1: aaa
$2: bbb
$3: ccc
$1: bbb  # $2だったものが$1に
$2: ccc  # $3だったものが$2に
```

### 動作の説明

- **shift前**: `$1=aaa`, `$2=bbb`, `$3=ccc`
- **shift後**: `$1=bbb`, `$2=ccc`（`aaa`は削除される）

> **ユースケース**: 多数の引数を順次処理する際に便利です。

# エスケープ文字とエスケープシーケンス

## エスケープ文字（`\`）

バックスラッシュ（`\`）は、直後の1文字の扱いを変更します。

### 行の継続

行末にバックスラッシュを付けることで、コマンドを複数行に分割できます。

```bash
$ echo "I am a cat. As yet I have no name."
I am a cat. As yet I have no name.

$ echo "I am a cat. \
> As yet I have no name."
I am a cat. As yet I have no name.
```

> **利点**: 長いコマンドの視認性を向上できます。

### 特殊文字のエスケープ

```bash
# ダブルクォートを出力する方法1: シングルクォートで囲む
$ echo 'Value of ABC is "$ABC".'
Value of ABC is "$ABC".

# ダブルクォートを出力する方法2: バックスラッシュでエスケープ
$ echo "Value of ABC is \"$ABC\"."
Value of ABC is "123".

# $記号をそのまま表示
$ echo "Value of ABC is \$ABC."
Value of ABC is $ABC.
```

## エスケープシーケンス

`echo -e`オプションを使用すると、エスケープシーケンスが有効になります。

### 主要なエスケープシーケンス

| シーケンス | 説明 |
|-----------|------|
| `\n` | 改行 |
| `\t` | タブ |
| `\r` | キャリッジリターン |
| `\\` | バックスラッシュ |
| `\0nnn` | 8進数で文字を指定（例: `\041`は`!`） |

### 使用例

```bash
$ echo -e "I am a cat. \nAs yet I have no name\041"
I am a cat.
As yet I have no name!
```

# sourceコマンド

## 概要

`source`コマンド（または`.`コマンド）は、指定されたファイルを現在のシェル環境で読み込んで実行します。

> **詳細**: 設定ファイルの永続化については[シェル変数.md](シェル変数.md)を参照してください。

## 通常実行とsourceの違い

### 通常実行（`./script.sh`）

サブシェルで実行されるため、スクリプト内の変数は親シェルに影響しません。

```bash
$ cat set.sh
#!/bin/bash
abc=xyz
echo $abc

$ echo $abc
# 何も表示されない（変数が未定義）

$ ./set.sh
xyz
# スクリプト内で設定された変数が出力される

$ echo $abc
# 何も表示されない（変数はサブシェル内でのみ有効）
```

### source実行

現在のシェル環境で実行されるため、変数が保持されます。

```bash
$ source set.sh
xyz

$ echo $abc
xyz
# sourceで読み込んだため、変数が現在のシェルに残る
```

## 主な用途

- 設定ファイル（`.bashrc`, `.bash_profile`など）の変更を即座に反映
- 共通の変数や関数を複数のスクリプトで共有

```bash
# 設定ファイルの変更を即座に反映
$ source ~/.bashrc

# 短縮形
$ . ~/.bashrc
```

# ベストプラクティス

## スクリプトの冒頭

```bash
#!/usr/bin/env bash

# 厳格モード
set -euo pipefail

# スクリプトの説明
# このスクリプトは...

# 定数定義
readonly SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
readonly LOG_FILE="/var/log/script.log"
```

## エラーハンドリング

```bash
#!/bin/bash

# エラーメッセージを標準エラー出力に
error() {
    echo "ERROR: $*" >&2
    exit 1
}

# 使用例
[ -f "$file" ] || error "File not found: $file"
```

## 変数の命名規則

```bash
# 定数: 大文字とアンダースコア
readonly MAX_RETRY=3
readonly CONFIG_FILE="/etc/app.conf"

# 変数: 小文字とアンダースコア
user_name="john"
file_count=0

# 環境変数: 大文字（慣習）
export PATH="/usr/local/bin:$PATH"
```

## クォートの使用

```bash
# 良い例: 変数を必ずクォート
if [ -f "$file" ]; then
    cp "$file" "$destination"
fi

# 悪い例: クォートなし（スペースを含むファイル名で問題）
if [ -f $file ]; then
    cp $file $destination
fi
```

# 実用的なスクリプトテンプレート

## 基本テンプレート

```bash
#!/usr/bin/env bash

set -euo pipefail

# スクリプトの説明
readonly SCRIPT_NAME="$(basename "$0")"
readonly SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

# 使用方法を表示
usage() {
    cat <<EOF
Usage: $SCRIPT_NAME [OPTIONS] <argument>

Description:
    このスクリプトの説明

Options:
    -h, --help      このヘルプを表示
    -v, --verbose   詳細出力
    -d, --debug     デバッグモード

Examples:
    $SCRIPT_NAME file.txt
    $SCRIPT_NAME -v file.txt
EOF
}

# エラーで終了
die() {
    echo "ERROR: $*" >&2
    exit 1
}

# メイン処理
main() {
    # 引数チェック
    if [ $# -eq 0 ]; then
        usage
        exit 1
    fi
    
    # 処理
    echo "Processing..."
}

# スクリプト実行
main "$@"
```

## オプション解析付きテンプレート

```bash
#!/usr/bin/env bash

set -euo pipefail

# デフォルト値
VERBOSE=0
DEBUG=0

# オプション解析
while [[ $# -gt 0 ]]; do
    case $1 in
        -h|--help)
            usage
            exit 0
            ;;
        -v|--verbose)
            VERBOSE=1
            shift
            ;;
        -d|--debug)
            DEBUG=1
            set -x
            shift
            ;;
        -*)
            echo "Unknown option: $1" >&2
            usage
            exit 1
            ;;
        *)
            # 位置引数
            FILE="$1"
            shift
            ;;
    esac
done

# メイン処理
main() {
    [ "$VERBOSE" -eq 1 ] && echo "Verbose mode enabled"
    [ "$DEBUG" -eq 1 ] && echo "Debug mode enabled"
    
    # 処理
    echo "Processing: $FILE"
}

main
```

# トラブルシューティング

## よくあるエラー

### 1. "command not found"

```bash
# 原因: PATHが通っていない、または実行権限がない
# 解決: フルパスで実行するか、権限を確認

$ chmod +x script.sh
$ ./script.sh
```

### 2. "Permission denied"

```bash
# 原因: 実行権限がない
# 解決: 実行権限を付与

$ chmod u+x script.sh
```

### 3. "bad interpreter"

```bash
# 原因: シェバンのパスが間違っている
# 解決: 正しいパスに修正

#!/usr/bin/env bash  # 推奨
```

### 4. "syntax error near unexpected token"

```bash
# 原因: 構文エラー（スペース、クォート、括弧など）
# 解決: ShellCheckで確認

$ shellcheck script.sh
```
