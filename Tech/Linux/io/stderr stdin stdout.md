# stderr/stdin/stdout

## 標準入力 (stdin) - ファイルディスクリプタ 0

キーボードからの入力プログラムへ

## 標準出力 (stdout) - ファイルディスクリプタ 1

成功したメッセージの表示画面（端末）

## 標準エラー出力 (stderr) - ファイルディスクリプタ 2

エラーメッセージの表示画面（端末）

---

## リダイレクト

出力先をファイルに切り替える仕組み。

| 記号 | 動作                                | 例                         |
| ---- | ----------------------------------- | -------------------------- |
| `>`  | 上書き（既存の中身が消える）        | `echo "hello" > file.txt`  |
| `>>` | 追記（既存の末尾に追加）            | `echo "world" >> file.txt` |
| `<`  | ファイルを標準入力として渡す        | `wc -l < file.txt`         |
| `2>` | 標準エラー出力をリダイレクト        | `cmd 2> error.log`         |
| `&>` | stdout と stderr を両方リダイレクト | `cmd &> all.log`           |

## ヒアドキュメント（Here Document）

`<<` を使い、複数行のテキストをまとめて標準入力として渡す構文。

```bash
コマンド << 終端文字列
  テキスト1行目
  テキスト2行目
終端文字列
```

終端文字列（デリミタ）は任意の文字列でよい。慣習的に `EOF` がよく使われる。

### クォートの有無による違い

| 書き方     | 変数展開 | 例                            |
| ---------- | -------- | ----------------------------- |
| `<< EOF`   | される   | `$HOME` → `/home/user`        |
| `<< 'EOF'` | されない | `$HOME` → `$HOME`（そのまま） |

### 実用例：.bashrc への追記

```bash
cat >> ~/.bashrc << 'EOF'
alias ls='eza --icons --group-directories-first'
eval "$(zoxide init bash)"
EOF
```

この例の分解:

- **`cat`** — テキストを受け取って出力する
- **`>>`** — その出力を `~/.bashrc` の末尾に追記する
- **`<< 'EOF'`** — 入力は次の `EOF` 行までの複数行テキスト（変数展開なし）

`'EOF'` とクォートしているのは、`$(zoxide init bash)` を展開せずそのまま文字列として書き込みたいため。
