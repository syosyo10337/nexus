# シェルスクリプトのデバッグ

> **関連ドキュメント**: シェルスクリプトの基礎については[シェルスクリプト.md](シェルスクリプト.md)を参照してください。

シェルスクリプトのデバッグ手法とエラーハンドリングについて説明します。

---

## shコマンドによるデバッグ

`sh -x`または`bash -x`オプションを使用すると、コマンドや変数の内容を表示しながらスクリプトを実行できます。

### 基本構文

```bash
sh -x スクリプト名
# または
bash -x スクリプト名
```

### 実行例

スクリプト（`sample.sh`）:

```bash
#!/bin/bash
name="John"
age=30
echo "Name: $name"
echo "Age: $age"
```

デバッグ実行:

```bash
$ bash -x sample.sh
+ name=John
+ age=30
+ echo 'Name: John'
Name: John
+ echo 'Age: 30'
Age: 30
```

> **解説**: `+`記号の後に実際に実行されるコマンドが表示されます。変数は展開された値が表示されます。

### 複雑なスクリプトの例

```bash
$ bash -x ./sample.sh
+ true
+ echo "Continue? (y/n)"
Continue? (y/n)
+ read input
y
+ case $input in
+ continue
+ true
+ echo "Continue? (y/n)"
Continue? (y/n)
+ read input
n
+ case $input in
+ break
```

## スクリプト内でのデバッグモード

### set -x / set +x

スクリプト内で動的にデバッグモードを切り替えられます。

```bash
#!/bin/bash

echo "Normal execution"

# デバッグモードを有効化
set -x

name="John"
age=30
echo "Name: $name"

# デバッグモードを無効化
set +x

echo "Back to normal"
```

出力:

```
Normal execution
+ name=John
+ age=30
+ echo 'Name: John'
Name: John
+ set +x
Back to normal
```

### 部分的なデバッグ

```bash
#!/bin/bash

# 通常の処理
initialize_variables() {
    var1="value1"
    var2="value2"
}

# デバッグしたい部分のみ
debug_section() {
    set -x
    # ここだけデバッグ出力される
    local result=$(complex_calculation)
    process_data "$result"
    set +x
}

initialize_variables
debug_section
```

## setオプション

### 主要なデバッグオプション

| オプション | 説明 |
|-----------|------|
| `set -x` | コマンドを実行前に表示（xtrace） |
| `set -v` | コマンドを読み込み時に表示（verbose） |
| `set -e` | エラーが発生したら即座にスクリプトを終了 |
| `set -u` | 未定義の変数を参照したらエラー |
| `set -o pipefail` | パイプライン内のコマンドが失敗したらエラー |

### set -e（エラーで終了）

```bash
#!/bin/bash
set -e

echo "Step 1"
false  # このコマンドは失敗する
echo "Step 2"  # ここは実行されない
```

出力:

```
Step 1
```

> **注意**: `set -e`は条件文内では無効化されます。

### set -u（未定義変数のエラー）

```bash
#!/bin/bash
set -u

name="John"
echo "Name: $name"
echo "Age: $age"  # エラー: ageは未定義
```

出力:

```
Name: John
./script.sh: line 5: age: unbound variable
```

### set -o pipefail（パイプラインのエラー）

```bash
#!/bin/bash

# デフォルト動作（最後のコマンドの終了ステータスのみ）
false | true
echo "Exit code: $?"  # 0（trueの終了ステータス）

# pipefailを有効化
set -o pipefail
false | true
echo "Exit code: $?"  # 1（falseの終了ステータス）
```

## 厳格モード（Strict Mode）

### 推奨される設定

```bash
#!/bin/bash

# 厳格モード（エラーを見逃さない）
set -euo pipefail

# または個別に
# set -e          # エラーで終了
# set -u          # 未定義変数でエラー
# set -o pipefail # パイプラインのエラーを検出

# 以降の処理
echo "Starting script..."
```

### IFS（Internal Field Separator）の設定

```bash
#!/bin/bash

# より安全な設定
set -euo pipefail
IFS=$'\n\t'

# スクリプトの処理
```

> **解説**: IFSをデフォルトのスペース・タブ・改行から改行・タブのみに変更することで、スペースを含むファイル名の扱いが安全になります。

## エラーハンドリング

### 終了ステータスの確認

```bash
#!/bin/bash

# 方法1: if文で確認
if command; then
    echo "Success"
else
    echo "Failed"
fi

# 方法2: ||演算子
command || echo "Failed"

# 方法3: &&演算子
command && echo "Success"

# 方法4: 終了ステータスを変数に保存
command
status=$?
if [ $status -ne 0 ]; then
    echo "Command failed with status: $status"
fi
```

### エラーメッセージの出力

```bash
#!/bin/bash

# 標準エラー出力に出力
echo "Error: File not found" >&2

# 関数化
error() {
    echo "ERROR: $*" >&2
}

error "Something went wrong"
```

### エラーハンドリング関数

```bash
#!/bin/bash

# エラーで終了する関数
die() {
    echo "ERROR: $*" >&2
    exit 1
}

# 使用例
[ -f "$config_file" ] || die "Config file not found: $config_file"
```

## トラップ（trap）

### 基本構文

```bash
trap 'コマンド' シグナル
```

### クリーンアップ処理

```bash
#!/bin/bash

# 一時ファイルの作成
temp_file=$(mktemp)

# スクリプト終了時に一時ファイルを削除
trap "rm -f $temp_file" EXIT

# 処理
echo "Working with temp file: $temp_file"
# ...

# EXITトラップが自動的に実行される
```

### エラー時の処理

```bash
#!/bin/bash

cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/lockfile
    # その他のクリーンアップ処理
}

# エラー時にクリーンアップ
trap cleanup ERR EXIT

set -e

# 処理
echo "Starting..."
false  # エラーが発生
echo "This won't be executed"
```

### 主要なシグナル

| シグナル | 説明 |
|---------|------|
| `EXIT` | スクリプト終了時 |
| `ERR` | コマンドがエラーを返した時 |
| `INT` | Ctrl+C（割り込み） |
| `TERM` | 終了シグナル |
| `DEBUG` | 各コマンド実行前 |

### 複数のシグナルをキャッチ

```bash
#!/bin/bash

cleanup() {
    echo "Interrupted. Cleaning up..."
    # クリーンアップ処理
    exit 1
}

trap cleanup INT TERM

echo "Running... (Press Ctrl+C to stop)"
while true; do
    sleep 1
done
```

## デバッグ用の関数

### デバッグ出力関数

```bash
#!/bin/bash

DEBUG=1  # デバッグモードを有効化

debug() {
    if [ "$DEBUG" = "1" ]; then
        echo "DEBUG: $*" >&2
    fi
}

# 使用例
debug "Starting process"
name="John"
debug "Name is set to: $name"
```

### アサーション関数

```bash
#!/bin/bash

assert() {
    if ! "$@"; then
        echo "Assertion failed: $*" >&2
        exit 1
    fi
}

# 使用例
assert [ -f "/etc/passwd" ]
assert [ "$USER" != "root" ]
```

### スタックトレース

```bash
#!/bin/bash

print_stack_trace() {
    local frame=0
    while caller $frame; do
        ((frame++))
    done
}

error_with_trace() {
    echo "ERROR: $*" >&2
    echo "Stack trace:" >&2
    print_stack_trace >&2
    exit 1
}

# 使用例
function_a() {
    function_b
}

function_b() {
    function_c
}

function_c() {
    error_with_trace "Something went wrong"
}

function_a
```

## ログ出力

### 基本的なログ関数

```bash
#!/bin/bash

log() {
    local level="$1"
    shift
    local message="$*"
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[$timestamp] [$level] $message"
}

# 使用例
log "INFO" "Application started"
log "ERROR" "Failed to connect to database"
log "DEBUG" "Variable value: $variable"
```

### ログレベル付き

```bash
#!/bin/bash

LOG_LEVEL=${LOG_LEVEL:-INFO}

log_debug() {
    [ "$LOG_LEVEL" = "DEBUG" ] && log "DEBUG" "$*"
}

log_info() {
    log "INFO" "$*"
}

log_error() {
    log "ERROR" "$*" >&2
}

log() {
    local level="$1"
    shift
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
}

# 使用例
log_debug "Detailed information"
log_info "Normal operation"
log_error "Something went wrong"
```

### ファイルへのログ出力

```bash
#!/bin/bash

LOG_FILE="/var/log/myscript.log"

log() {
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[$timestamp] $*" | tee -a "$LOG_FILE"
}

# 使用例
log "Script started"
log "Processing files..."
log "Script completed"
```

## 実行時間の計測

### 基本的な計測

```bash
#!/bin/bash

start_time=$(date +%s)

# 処理
sleep 2

end_time=$(date +%s)
elapsed=$((end_time - start_time))
echo "Elapsed time: ${elapsed}s"
```

### より精密な計測

```bash
#!/bin/bash

# ナノ秒単位で計測
start_time=$(date +%s%N)

# 処理
sleep 0.5

end_time=$(date +%s%N)
elapsed=$(( (end_time - start_time) / 1000000 ))
echo "Elapsed time: ${elapsed}ms"
```

### 関数の実行時間計測

```bash
#!/bin/bash

time_function() {
    local func_name="$1"
    shift
    local start=$(date +%s%N)
    
    "$func_name" "$@"
    
    local end=$(date +%s%N)
    local elapsed=$(( (end - start) / 1000000 ))
    echo "Function $func_name took ${elapsed}ms" >&2
}

# 使用例
my_function() {
    sleep 1
    echo "Done"
}

time_function my_function
```

## ShellCheck

### ShellCheckとは

静的解析ツールで、スクリプトの問題を事前に検出できます。

### インストール

```bash
# macOS
brew install shellcheck

# Ubuntu/Debian
apt-get install shellcheck

# CentOS/RHEL
yum install shellcheck
```

### 使用方法

```bash
# スクリプトをチェック
shellcheck script.sh

# より厳格なチェック
shellcheck -x script.sh

# 特定の警告を無視
shellcheck -e SC2086 script.sh
```

### スクリプト内で警告を無効化

```bash
#!/bin/bash

# shellcheck disable=SC2086
echo $variable

# 複数の警告を無効化
# shellcheck disable=SC2086,SC2034
```

## デバッグのベストプラクティス

### 1. 早期のエラー検出

```bash
#!/bin/bash
set -euo pipefail

# スクリプトの処理
```

### 2. 明確なエラーメッセージ

```bash
# 悪い例
[ -f "$file" ] || exit 1

# 良い例
[ -f "$file" ] || {
    echo "ERROR: File not found: $file" >&2
    exit 1
}
```

### 3. ログ出力の活用

```bash
#!/bin/bash

log_info "Starting backup process"
log_debug "Source: $source, Destination: $dest"

if backup_files; then
    log_info "Backup completed successfully"
else
    log_error "Backup failed"
    exit 1
fi
```

### 4. 段階的なテスト

```bash
#!/bin/bash

# 開発中はドライランモード
DRY_RUN=1

execute() {
    if [ "$DRY_RUN" = "1" ]; then
        echo "DRY RUN: $*"
    else
        "$@"
    fi
}

# 使用例
execute rm -rf /important/directory
```
