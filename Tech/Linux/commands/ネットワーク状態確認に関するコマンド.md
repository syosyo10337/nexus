# ネットワーク状態確認に関するコマンド

- ping

```Bash
$ ping 宛先ホスト名or宛先IPアドレス
```

手元の環境からAWS上のwebサーバーに到達できるかや、webサーバーとDBサーバー間で通信ができるかを調べることができる。Ctrl + c で終了。

応答速度が見れます。（ICMPプロトコルを使っているインターネット層）

- traceroute

```Bash
$ traceroute URL/
```

こちらもICMPを使ったコマンド。宛先へのネットワーク上の経路を確認するために利用するコマンドです。`*`表示されるのは、ICMPを受け付けない構成になっていることを示す。

ルーティングを設定しているが、予想以上に時間がかかっているときなど経路確認的な意味合いで用いると良い。

- telnet

```Bash
$ telnet 宛先ホスト名or宛先IPアドレス ポート番号
# ポート番号を省略すると、telnetコマンドのwellknownポート23番が使われる。
```

TCP到達性の確認、つまりpingなどで疎通確認はできたものの、WEBサーバーのミドルウェアの起動確認や、ファイアウォールなどでのフィルタリングがされていないかなど、特定のポートへの到達確認するために使える。

実行されると、コマンド待受画面となり、接続できることがわかる。

- nslookup

```Bash
$ nslookup ホスト名またはIPaddress
# name server lookupのこと

$ dig
# こちらの方が今時らしい
```

### プログラムの待ち受けポートを確認する

list of open file コマンドを使用する。

`[LISTEN]` と書かれているのが、他のコンピュータから通信を待ち受けているポートを表し、`[ESTABLISHED]` が相手と通信中のポートを表す。

```Bash
$ sudo lsof -i -n
# -i 「-i」で全てのネットワークソケットを対象にする。「-i4」でIPv4、「-i6」でIPv6を指定できる。この他、「-iTCP」「-iUDP」「-i@ホスト」「-i:ポート番号」を指定可能。
# -p <プロセスID> 表示対象にしたいプロセスIDを指定する
# -n ホスト名の代わりにIPアドレスを表示する
# -P ポート名の代わりにポート番号を表示する

# e.g.)
# COMMAND     PID    USER   FD   TYPE             DEVICE SIZE/OFF NODE NAME
# loginwind   162 masanao    9u  IPv4 0x7c3a80436cbfc923      0t0  UDP *:*
# rapportd    551 masanao    3u  IPv4 0x7c3a8039d1fd5a6b      0t0  TCP *:64665 (LISTEN)
# rapportd    551 masanao    4u  IPv6 0x7c3a8039d4050463      0t0  TCP *:64665 (LISTEN)
# rapportd    551 masanao    5u  IPv4 0x7c3a80436cbdd923      0t0  UDP *:*
# rapportd    551 masanao    6u  IPv4 0x7c3a80436cbd8923      0t0  UDP *:*
# rapportd    551 masanao   10u  IPv4 0x7c3a8039d0d6ffc3      0t0  TCP 192.168.11.7:64665->192.168.11.4:58585 (ESTABLISHED)
```
