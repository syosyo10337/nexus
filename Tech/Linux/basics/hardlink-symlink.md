# ハードリンクとシンボリックリンク

## リンクとは

リンクとは、ファイルに別名をつける機能です。Linuxには**ハードリンク**と**シンボリックリンク**の2種類があります。

### ハードリンク

1つのファイルの実体に複数の名前をつける機能。

**制約:**

- ディレクトリに対して作成できない
- 別のファイルシステムをまたいで作成できない

**重要な特徴:**

- ファイルの実体がなくなるのは、すべてのハードリンクが削除された時

### シンボリックリンク

リンク先のパス名が書かれた小さな特殊ファイル。ショートカットやエイリアスに似た機能。

**特徴:**

- リンク先がファイルの実体
- 実体だけ削除してリンク先だけ残っているとリンクが壊れた状態になる

## ファイルの実体を理解する

ファイルは実は2つの部分でできています：

```text
「ファイル名」 → 「実データ（inode）」
```

```text
┌──────────┐       ┌─────────────────┐
│ lodash.js │ ───→  │ ディスク上の実体  │
│ (名前)    │       │ (inode #12345)   │
└──────────┘       └─────────────────┘
```

ファイル名は「実データへのポインタ（参照）」に過ぎないのです。

## コピー vs シンボリックリンク vs ハードリンク

### コピー（cp）

```text
┌──────────┐       ┌─────────────────┐
│ fileA.js  │ ───→  │ 実データ①        │  ← ディスク容量 2倍
└──────────┘       └─────────────────┘
┌──────────┐       ┌─────────────────┐
│ fileB.js  │ ───→  │ 実データ②(複製)  │
└──────────┘       └─────────────────┘
```

### シンボリックリンク（ln -s）= ショートカット

```text
┌──────────┐       ┌─────────────────┐
│ fileA.js  │ ───→  │ 実データ①        │
└──────────┘       └─────────────────┘
┌──────────┐    ↗
│ fileB.js  │ → fileA.js              ← 「fileAを見てね」という間接参照
└──────────┘
```

### ハードリンク（ln）= もう一つの名前

```text
┌──────────┐ ─┐    ┌─────────────────┐
│ fileA.js  │  ├──→ │ 実データ①        │  ← 同じ実体を直接指す
└──────────┘  │    └─────────────────┘
┌──────────┐ ─┘
│ fileB.js  │
└──────────┘
```

## ハードリンクの特徴

### 実際に試してみる

```bash
# 実際にやってみると
echo "hello" > fileA.js
ln fileA.js fileB.js        # ハードリンク作成

cat fileA.js  # → hello
cat fileB.js  # → hello     （同じ実データ）

# fileAを消しても...
rm fileA.js
cat fileB.js  # → hello     （まだ読める！実データは残っている）
```

### まとめ

- **ディスク容量を消費しない**（実データは1つだけ）
- **片方を消してももう片方は残る**（参照カウントが0になるまで実データは消えない）
- **シンボリックリンクと違い、リンク元が消えても壊れない**

## 実用例：パッケージマネージャーでの活用

### npm（コピー方式）

```text
プロジェクトA/node_modules/lodash.js  → 実データ① (5MB)
プロジェクトB/node_modules/lodash.js  → 実データ② (5MB)
合計: 10MB
```

### pnpm（ハードリンク方式）

```text
store/v10/xxxx                        → 実データ① (5MB)
プロジェクトA/node_modules/lodash.js  ─┤
プロジェクトB/node_modules/lodash.js  ─┘ （全部同じ実データ）
合計: 5MB
```

10個のプロジェクトで同じパッケージを使っても、ディスク上の実体は1つだけ。これが pnpm が速くてディスクに優しい理由です。

## ln コマンドの使い方

### ハードリンクの作成

```bash
ln [リンク元ファイル名] [リンク名]
```

### シンボリックリンクの作成

```bash
ln -s [リンク元ファイル名] [リンク名]
```

### オプション

- `-s`: シンボリックリンクを作成
- `-f`: 上書き（force）
- `-i`: 上書き前に確認
