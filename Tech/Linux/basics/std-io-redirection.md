# 標準入出力とリダイレクト

## 出力のリダイレクト re-direct: `>`
次のコマンドはls の結果を出力をリダイレクトして、ファイルに書き込む例です。`>` の記号を用いることでリダイレクトができます。

```bash
# e.g.
ls > ls-output
```

コマンドを実行するとls-output というファイルが作成され、リダイレクトするため、標準出力先を変更しファイルに出力を格納している。上書きせず追記したい場合は、アペンド(`>>`) の記号を用います。

**catコマンドによるファイル作成**
標準出力をリダイレクトすることでファイルを作成できます。cat コマンドとリダイレクトを組み合わせると、自由な内容でファイルを作成することができます。

```bash
# e.g. 
cat > [つけたいファイル名]
[ファイルの中身]
ctrl+d
```

`ctrl+d`でEOF(end of file)してデータの入力を終了する。Linuxではデータの読み込みが最後になると、このEOF が入力され終わる決まりがあります。cat はEOFを受け取ったことで入力が終わったと判断し、リダイレクトを終了します。

## 標準エラー出力のリダイレクト: `2>`
リダイレクトをつかって前の内容を `>` 後に出力できますが、`2>` とすることで標準エラー出力を表すので、エラーメッセージをファイルに記録できます。

ex)

```bash
ls /xxxx 2> list.txt
```

標準出力と標準エラー出力を別々のファイルに出力したい場合：

```bash
ls /xxx > list.txt 2> error.txt
```

画面には何も表示されず、実行結果は `list.txt` へ、エラーメッセージは `error.txt` へ出力される。

**標準出力と標準エラー出力を1つのファイルにまとめる**
標準出力と、標準エラー出力を１つのファイルに出力したい場合は、標準出力をリダイレクトした後ろに `2>&1` を付けます。

ex)

```bash
ls /xxxx > result.txt 2>&1
```

または、より簡潔な記法として `&>` を使うこともできます：

```bash
ls /xxxx &> result.txt
```

追記する場合は `&>>` を使います：

```bash
ls /xxxx &>> result.txt
```

## リダイレクト演算子のまとめ

| 演算子 | 説明 |
|--------|------|
| `>` | 標準出力をファイルにリダイレクト（上書き） |
| `>>` | 標準出力をファイルに追記 |
| `2>` | 標準エラー出力をファイルにリダイレクト（上書き） |
| `2>>` | 標準エラー出力をファイルに追記 |
| `&>` | 標準出力と標準エラー出力の両方をファイルにリダイレクト（上書き） |
| `&>>` | 標準出力と標準エラー出力の両方をファイルに追記 |
| `2>&1` | 標準エラー出力を標準出力にリダイレクト |

**/dev/null**
スペシャルファイル。入力先としても何も返ってこないし、出力にしても書き込まれたデータは保存されて消えてなくなる。標準入力を空っぽにしたり、標準出力を捨てる時に使えます。

**フィルタ**
標準入力を入力として受け取り標準出力を出力するコマンドのこと。(ex: `cat`, `head`, `tail`, `grep`, `sort`...)
