# シェル変数

> **関連ドキュメント**: エイリアスについては[エイリアス.md](エイリアス.md)を参照してください。

## シェル変数とは

シェル変数とは、bash内部で使用される変数です。代表的なものに`PATH`、`LANG`、`PS1`(プロンプトの表示文字を保存する変数)などがあります。

## 変数の設定

```bash
# 基本書式
<変数名>=<入れたい値>

# 例
var1='test variable'
```

### 注意点

- 値にスペースが含まれる場合は、シングルクォート(`'`)またはダブルクォート(`"`)で囲む
- `=`の前後にスペースを入れない

## 変数の参照

```bash
# 基本書式
$<変数名>

# 例: シェル変数$var1をechoで出力
# echo: 引数にとった文字列を標準出力するコマンド
echo $var1
```

## 主要なシェル変数

### PATH

コマンドの実体ファイルを検索するディレクトリパスのリストを格納します。シェルがコマンドを実行する際、この変数に含まれるディレクトリから実行ファイルを探します。

自分でインストールしたアプリケーションのコマンドも、`PATH`に追加することでフルパスを指定せずに実行できるようになります。

```bash
# PATHの内容を確認（ディレクトリパスが:で連結されている）
echo $PATH
# 出力例:
# /Users/masanao/.nodebrew/current/bin:/usr/local/opt/mysql/bin:/Users/masanao/.rbenv/shims:/Users/masanao/.rbenv/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/Library/Apple/usr/bin
```

### LANG

ロケール(locale)を格納する変数です。言語や国、地域を特定する識別子が設定されています。

### HOME

ユーザーのホームディレクトリのパスを格納します。

```bash
# HOMEを参照して、その配下のDesktopディレクトリに移動
cd $HOME/Desktop
```

### SHELL

現在のログインシェルのパスを格納します。

### PWD

カレントディレクトリ(現在の作業ディレクトリ)のパスを保持します。

```bash
# pwdコマンドは内部的に$PWDを参照している
pwd
# /Users/masanao/Desktop/environment/recorda_me

echo $PWD
# /Users/masanao/Desktop/environment/recorda_me
```

### BASH_SOURCE

**Bashスクリプト内で使用される配列変数**で、現在実行中のスクリプトファイルのパスを格納します。

```bash
# スクリプト内で自身のパスを取得
echo "${BASH_SOURCE[0]}"

# スクリプトが配置されているディレクトリを取得
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

# シンボリックリンクを解決した絶対パスを取得する場合は -P を付ける
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd -P)"
```

#### pwd -P オプション

`pwd -P` はシンボリックリンクを解決した物理パスを返す。`-P` がないとリンク自体のパスが返る場合がある。

```bash
# 例: /opt/app -> /Users/masanao/projects/app のシンボリックリンクがある場合
cd /opt/app
pwd      # /opt/app（論理パス）
pwd -P   # /Users/masanao/projects/app（物理パス）
```

#### スクリプトのルートディレクトリを取得するパターン

スクリプトの配置ディレクトリから親ディレクトリ（リポジトリルートなど）を取得するイディオム:

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd -P)"
DOTDIR="$(dirname "${SCRIPT_DIR}")"
```

処理の流れ（内側から順に）:

1. `${BASH_SOURCE[0]}` — 実行中スクリプトのパス
2. `dirname "..."` — ディレクトリ部分を取得
3. `cd "..." && pwd -P` — そのディレクトリに移動し、シンボリックリンクを解決した絶対パスを取得
4. `dirname "${SCRIPT_DIR}"` — 一つ上の親ディレクトリを取得

具体例（`/Users/masanao/dotfiles/.bin/installer.sh` の場合）:

| 変数       | 値                             |
| ---------- | ------------------------------ |
| SCRIPT_DIR | /Users/masanao/dotfiles/.bin   |
| DOTDIR     | /Users/masanao/dotfiles        |

スクリプトがどこから実行されても（`cd /tmp && bash ~/dotfiles/.bin/installer.sh` など）、正しくリポジトリのルートを指せる。

#### BASH_SOURCEの特徴

- `${BASH_SOURCE[0]}` - 現在実行中のスクリプトファイルのパス
- `${BASH_SOURCE[1]}` - 呼び出し元のスクリプトファイルのパス（ネストしている場合）
- `source`コマンドや`.`コマンドでスクリプトを読み込んだ場合でも正しいパスを取得できる

#### $0との違い

- `$0` - シェルやスクリプトの名前（`source`で読み込まれた場合、親シェルの名前になる）
- `${BASH_SOURCE[0]}` - 常に現在のスクリプトファイルのパス

```bash
# 例: スクリプトの絶対パスを取得する実用的なパターン
#!/bin/bash
SCRIPT_PATH="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)/$(basename "${BASH_SOURCE[0]}")"
echo "このスクリプトのフルパス: $SCRIPT_PATH"
```

> **ユースケース**: スクリプトの配置場所に依存しない相対パス指定や、設定ファイルの読み込みなどに活用されます。

# 環境変数(Environment Variable)

環境変数は、シェルの外部コマンドからも参照できる変数です。

シェル変数はシェル内部でのみ使用されますが、環境変数は子プロセス(外部コマンド)にも引き継がれます。`LANG`のように、デフォルトで環境変数として設定されているものもあります。

## 環境変数の動作例

```bash
# 外部コマンドcatが環境変数LANGを参照して表示言語を切り替える例

# 日本語ロケールに設定
LANG='ja_JP.UTF-8'
cat --help
# 出力: 使用法: cat [オプション]...

# 英語ロケールに設定
LANG='en_US.UTF-8'
cat --help
# 出力: Usage: cat [option]...
```

## 外部コマンドと組み込みコマンド

### 外部コマンド

実行ファイルとしてファイルシステム上に存在するコマンド（例: `/bin/cp`, `/bin/ls`）

### 組み込みコマンド

シェル自体に内蔵されているコマンド（例: `cd`, `set`, `export`）

### コマンドの種類を確認する

```bash
# typeコマンドで確認

type cp
# 出力: cp is /bin/cp (外部コマンド)

type set
# 出力: set is a shell builtin (組み込みコマンド)

type type
# 出力: type is a shell builtin (組み込みコマンド)
```

## 環境変数に関するコマンド

### printenv

シェルに設定されている環境変数を一覧表示します。

```bash
printenv
# 出力例:
# TERM_PROGRAM=Apple_Terminal
# SHELL=/bin/zsh
# TERM=xterm-256color
# TMPDIR=/var/folders/c7/z8jt7r8n3gx7z4f06lvbhy040000gn/T/
# TERM_PROGRAM_VERSION=445
# TERM_SESSION_ID=F673BC44-F7A6-423E-9729-9F8D83F36B7F
# USER=masanao
# SSH_AUTH_SOCK=/private/tmp/com.apple.launchd.svka5cNefj/Listeners
# PATH=/Users/masanao/.nodebrew/current/bin:/usr/local/opt/mysql/bin:...
# __CFBundleIdentifier=com.apple.Terminal
# PWD=/Users/masanao/Desktop/environment/recorda_me
# XPC_FLAGS=0x0
```

### export

シェル変数を環境変数として設定します。

> **重要**: `=`で定義したシェル変数は、新しく起動したシェル(子プロセス)からは利用できません。外部コマンドから参照させるには、環境変数に設定する必要があります。

#### 基本的な使い方

```bash
# 方法1: シェル変数を定義してから環境変数に設定
LESS='--no-init'
export LESS

# 方法2: 定義と環境変数設定を同時に行う（推奨）
export LESS='--no-init'
```

#### 実用例

```bash
# lessコマンド用の環境変数を設定
# LESS変数には、lessコマンドに自動で付与したいオプション値を格納できる
# --no-init: lessコマンド終了時に画面内容を残す

# シェル変数に値を格納
LESS='--no-init'

# 環境変数として設定（$は不要）
export LESS

# または一行で
export LESS='--no-init'
```

# Bashの設定を永続化

端末を終了すると、変更した設定がリセットされてしまいます。次回ログイン時にカスタム設定を保持するには、**シェルの設定ファイルを編集する必要があります**。

## シェル起動時に読み込まれる設定ファイル

Bashの起動時には、以下の順序で設定ファイルが読み込まれます:

1. ユーザーがログインした際に`/etc/profile`が読み込まれる
2. 次に`~/.bash_profile`(zshの場合は`~/.zsh_profile`)が読み込まれる
3. 多くの場合、`~/.bash_profile`から`~/.bashrc`を呼び出す設定になっている

### 主要な設定ファイル

#### /etc/profile

- **用途**: システム全体で使用する設定ファイル
- **対象**: 全ユーザー共通の設定
- **編集**: 管理者権限が必要

#### ~/.bash_profile (または ~/.zsh_profile)

- **用途**: 各ユーザー個別の設定ファイル
- **読み込みタイミング**: ログイン時のみ
- **特徴**: 多くのディストリビューションでは、このファイルから`~/.bashrc`を呼び出す設定になっている

#### ~/.bashrc (または ~/.zshrc)

- **用途**: シェルの設定を記述
- **読み込みタイミング**: 
  - ログインシェル起動時（`~/.bash_profile`経由）
  - 非ログインシェル起動時（`bash`コマンドなどでシェルを起動した場合）

## 設定ファイルの記述に関するベストプラクティス

### 推奨事項

`/etc/profile`や`~/.bash_profile`はログイン時のみ読み込まれるため、**通常の設定変更は`~/.bashrc`(または`~/.zshrc`)に記述することを推奨します**。

### 設定変更時の注意点

1. **バックアップを作成する**

```bash
cp ~/.zshrc ~/.zshrc.backup
```

2. **予備のシェルを起動しておく**
   - 設定ミスでシェルが起動しなくなった場合に備える

3. **コメントを活用する**

```bash
# 複雑な設定にはコメントでメモを残す
export JAVA_HOME=/usr/lib/jvm/java-11  # Java 11のパス
```

## sourceコマンド

設定ファイルを編集した後、通常は再ログインが必要ですが、`source`コマンドを使うと即座に変更を反映できます。

```bash
# .bashrcの変更を即時反映
source ~/.bashrc

# または短縮形
. ~/.bashrc
```

## PATHを通す実用例

### 一時的にPATHを追加

```bash
# 現在のPATHに~/binを追加
PATH="$PATH:~/bin"
```

### 永続的にPATHを追加

```bash
# .zshrcに設定を追記（推奨）
echo 'export PATH="$ANDROID_HOME/platform-tools:$PATH"' >> ~/.zshrc

# 変更を即座に反映
source ~/.zshrc
```

> **注意**: `export`だけでは一時的な設定になります。永続化するには設定ファイル(`~/.zshrc`や`~/.bashrc`)に記述する必要があります。
