# 関数（サブルーチン）

一連の処理をまとめて、再利用可能にする方法について説明します。

## 関数の定義

### 基本構文

```bash
# 書き方1: functionキーワードを使用
function 関数名 {
    処理
}

# 書き方2: ()を使用（POSIX互換、推奨）
関数名() {
    処理
}

# 書き方3: functionキーワードと()の両方（冗長、非推奨）
function 関数名() {
    処理
}
```

> **推奨**: POSIX互換性のため、`関数名() { ... }`の形式を推奨します。

### 基本的な使用例

```bash
#!/bin/bash

# 関数の定義
greet() {
    echo "Hello, World!"
}

# 関数の呼び出し
greet
```

出力:

```
Hello, World!
```

## 引数の受け渡し

関数は、スクリプトと同様に位置パラメータ（`$1`, `$2`, ...）で引数を受け取ります。

### 基本的な引数の使用

```bash
#!/bin/bash

greet() {
    echo "Hello, $1!"
}

greet "World"
greet "Bash"
```

出力:

```
Hello, World!
Hello, Bash!
```

### 複数の引数

```bash
#!/bin/bash

add() {
    local result=$(($1 + $2))
    echo $result
}

sum=$(add 10 20)
echo "Sum: $sum"
```

出力:

```
Sum: 30
```

### 引数の数を確認

```bash
#!/bin/bash

print_args() {
    echo "引数の数: $#"
    echo "すべての引数: $@"
    
    local i=1
    for arg in "$@"
    do
        echo "引数$i: $arg"
        i=$((i + 1))
    done
}

print_args apple banana orange
```

出力:

```
引数の数: 3
すべての引数: apple banana orange
引数1: apple
引数2: banana
引数3: orange
```

## ローカル変数とグローバル変数

### グローバル変数（デフォルト）

関数内で定義した変数は、デフォルトでグローバルスコープになります。

```bash
#!/bin/bash

set_name() {
    name="John"  # グローバル変数
}

set_name
echo "Name: $name"  # Johnが表示される
```

### ローカル変数

`local`キーワードを使用すると、関数内でのみ有効な変数を定義できます。

```bash
#!/bin/bash

set_name() {
    local name="John"  # ローカル変数
    echo "Inside function: $name"
}

name="Alice"
set_name
echo "Outside function: $name"  # Aliceが表示される
```

出力:

```
Inside function: John
Outside function: Alice
```

> **ベストプラクティス**: 関数内の変数は基本的に`local`を使用することを推奨します。

## 戻り値

### return（終了ステータス）

`return`コマンドは、0-255の整数値（終了ステータス）を返します。

```bash
#!/bin/bash

check_file() {
    if [ -f "$1" ]; then
        return 0  # 成功（真）
    else
        return 1  # 失敗（偽）
    fi
}

# 使用例
if check_file "/etc/passwd"; then
    echo "ファイルが存在します"
else
    echo "ファイルが存在しません"
fi
```

### 終了ステータスの慣習

| 値 | 意味 |
|----|------|
| `0` | 成功 |
| `1` | 一般的なエラー |
| `2` | 誤った使用法（引数エラーなど） |
| `126` | コマンドが実行できない |
| `127` | コマンドが見つからない |
| `128+n` | シグナルnで終了 |

### echo（文字列の返却）

文字列を返す場合は、`echo`を使用してコマンド置換で受け取ります。

```bash
#!/bin/bash

get_message() {
    echo "This is a message"
}

# コマンド置換で値を取得
result=$(get_message)
echo "Result: $result"
```

出力:

```
Result: This is a message
```

### 複数の値を返す

```bash
#!/bin/bash

get_user_info() {
    echo "John Doe 30 Engineer"
}

# 複数の変数に分割して代入
read name1 name2 age job <<< $(get_user_info)
echo "Name: $name1 $name2"
echo "Age: $age"
echo "Job: $job"
```

## 実用的な関数の例

### ログ出力関数

```bash
#!/bin/bash

log() {
    local level="$1"
    shift
    local message="$@"
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[$timestamp] [$level] $message"
}

log "INFO" "Application started"
log "ERROR" "Failed to connect to database"
log "DEBUG" "Variable value:" "$variable"
```

### エラーハンドリング関数

```bash
#!/bin/bash

error_exit() {
    echo "ERROR: $1" >&2
    exit "${2:-1}"
}

# 使用例
if [ ! -f "$config_file" ]; then
    error_exit "Config file not found: $config_file" 2
fi
```

### ファイル存在確認関数

```bash
#!/bin/bash

file_exists() {
    if [ -f "$1" ]; then
        return 0
    else
        echo "Error: File '$1' not found" >&2
        return 1
    fi
}

# 使用例
if file_exists "/etc/passwd"; then
    echo "File exists"
fi
```

### 入力検証関数

```bash
#!/bin/bash

is_number() {
    local re='^[0-9]+$'
    if [[ $1 =~ $re ]]; then
        return 0
    else
        return 1
    fi
}

# 使用例
read -p "Enter a number: " input
if is_number "$input"; then
    echo "Valid number: $input"
else
    echo "Invalid input"
fi
```

### リトライ関数

```bash
#!/bin/bash

retry() {
    local max_attempts="$1"
    shift
    local command="$@"
    local attempt=1
    
    until $command
    do
        if [ $attempt -ge $max_attempts ]; then
            echo "Failed after $max_attempts attempts"
            return 1
        fi
        echo "Attempt $attempt failed. Retrying..."
        sleep 2
        attempt=$((attempt + 1))
    done
    
    return 0
}

# 使用例
retry 3 curl -f https://example.com
```

## 関数のベストプラクティス

### 1. 単一責任の原則

関数は1つのことだけを行うべきです。

```bash
# 悪い例: 複数の責任
process_user() {
    validate_user "$1"
    save_to_database "$1"
    send_email "$1"
}

# 良い例: 責任を分割
validate_user() {
    # バリデーションのみ
}

save_user() {
    # データベース保存のみ
}

notify_user() {
    # 通知のみ
}
```

### 2. 明確な命名

関数名は、その動作を明確に表すべきです。

```bash
# 良い例
get_user_name() { ... }
is_valid_email() { ... }
calculate_total() { ... }
```

### 3. エラーチェック

引数の検証とエラーハンドリングを行います。

```bash
#!/bin/bash

divide() {
    if [ $# -ne 2 ]; then
        echo "Usage: divide <number1> <number2>" >&2
        return 2
    fi
    
    if [ "$2" -eq 0 ]; then
        echo "Error: Division by zero" >&2
        return 1
    fi
    
    echo $(($1 / $2))
    return 0
}
```

### 4. ドキュメント化

複雑な関数にはコメントを追加します。

```bash
#!/bin/bash

# ファイルをバックアップする
# 引数:
#   $1 - バックアップ元のファイルパス
#   $2 - バックアップ先のディレクトリ（オプション、デフォルト: /backup）
# 戻り値:
#   0 - 成功
#   1 - ファイルが存在しない
#   2 - バックアップに失敗
backup_file() {
    local source="$1"
    local dest_dir="${2:-/backup}"
    
    if [ ! -f "$source" ]; then
        echo "Error: Source file not found" >&2
        return 1
    fi
    
    if ! cp "$source" "$dest_dir/"; then
        echo "Error: Backup failed" >&2
        return 2
    fi
    
    echo "Backup successful: $source -> $dest_dir/"
    return 0
}
```

## 高度なテクニック

### 再帰関数

```bash
#!/bin/bash

# 階乗を計算する再帰関数
factorial() {
    local n=$1
    if [ $n -le 1 ]; then
        echo 1
    else
        local prev=$(factorial $((n - 1)))
        echo $((n * prev))
    fi
}

result=$(factorial 5)
echo "5! = $result"  # 120
```

### 関数のエクスポート

サブシェルでも関数を使用可能にします。

```bash
#!/bin/bash

my_function() {
    echo "This is my function"
}

# 関数をエクスポート
export -f my_function

# サブシェルで実行
bash -c 'my_function'
```

### 関数の上書き

```bash
#!/bin/bash

# オリジナルの関数
greet() {
    echo "Hello"
}

greet  # Hello

# 関数を上書き
greet() {
    echo "Hi there!"
}

greet  # Hi there!
```

### コマンドのラッパー関数

```bash
#!/bin/bash

# lsコマンドのラッパー
ls() {
    command ls --color=auto -lh "$@"
}

# grepコマンドのラッパー
grep() {
    command grep --color=auto "$@"
}
```

## 関数ライブラリの作成

### ライブラリファイルの作成

```bash
# lib/utils.sh
#!/bin/bash

# 文字列関数
to_upper() {
    echo "$1" | tr '[:lower:]' '[:upper:]'
}

to_lower() {
    echo "$1" | tr '[:upper:]' '[:lower:]'
}

# 日付関数
get_timestamp() {
    date '+%Y%m%d_%H%M%S'
}

# ファイル関数
create_backup() {
    local file="$1"
    local backup="${file}.$(get_timestamp).bak"
    cp "$file" "$backup"
    echo "$backup"
}
```

### ライブラリの読み込み

```bash
#!/bin/bash

# ライブラリを読み込む
source ./lib/utils.sh

# または
. ./lib/utils.sh

# 関数を使用
name=$(to_upper "john")
echo "Name: $name"

backup=$(create_backup "config.txt")
echo "Backup created: $backup"
```

## デバッグ

### 関数のトレース

```bash
#!/bin/bash

# 関数の実行をトレース
set -x

my_function() {
    local var="test"
    echo "Value: $var"
}

my_function

set +x
```

### 関数の存在確認

```bash
#!/bin/bash

# 関数が定義されているか確認
if declare -f my_function > /dev/null; then
    echo "Function exists"
else
    echo "Function not found"
fi

# または
if type my_function &> /dev/null; then
    echo "Function exists"
fi
```
