# エイリアス(Alias)

エイリアスを使用すると、よく使うコマンドに短縮名を付けたり、デフォルトオプションを設定できます。

## エイリアスの作成

```bash
# 基本書式
alias <エイリアス名>='<コマンド>'

# 例: lsコマンドに-Fオプションを常に付与
alias ls='ls -F'

# 例: よく使うコマンドに短縮名を設定
alias ll='ls -la'
alias gs='git status'
alias gp='git pull'
```

## エイリアスの確認

```bash
# コマンドがエイリアスかどうかを確認
type <コマンド名>

# 例: lsにエイリアスが設定されている場合
type ls
# 出力: ls is an alias for ls -a

# すべてのエイリアスを表示
alias
```

## エイリアスの削除

```bash
# 特定のエイリアスを削除
unalias <エイリアス名>

# 例
unalias ls

# すべてのエイリアスを削除
unalias -a
```

## エイリアスを一時的に無効にする方法

エイリアスを削除せずに、一時的に本来のコマンドを実行する方法:

### 1. フルパスで実行

```bash
/bin/<コマンド名>

# 例
/bin/ls
```

### 2. commandコマンドを使用

```bash
command <コマンド名>

# 例
command ls
```

### 3. バックスラッシュでエスケープ

```bash
\<コマンド名>

# 例
\ls
```

## エイリアスの永続化

エイリアスは、設定ファイルに記述することで永続化できます。

```bash
# ~/.bashrcまたは~/.zshrcに追記
echo "alias ll='ls -la'" >> ~/.zshrc

# 変更を即座に反映
source ~/.zshrc
```

## 注意点

1. **エイリアスはシェルスクリプト内では展開されない**
   - スクリプト内で使用する場合は、`shopt -s expand_aliases`を設定する必要がある

2. **既存のコマンドを上書きする場合は注意**
   - 意図しない動作を引き起こす可能性がある
   - 本来のコマンドが必要な場合は、上記の無効化方法を使用

3. **エイリアスは現在のシェルセッションのみ有効**
   - 永続化するには設定ファイルに記述する必要がある

## 関連コマンド

- `type` - コマンドの種類を確認
- `which` - コマンドの実行ファイルのパスを表示
- `command` - エイリアスを無視してコマンドを実行
- `source` - 設定ファイルを再読み込み

## 参考

エイリアスは便利ですが、過度に使用すると他の環境で作業する際に混乱する可能性があります。チーム開発では、共通のエイリアス設定を共有することを検討してください。
