# ベストプラクティス

> **関連**: [シェルスクリプト.md](シェルスクリプト.md)｜[スクリプトの作成と実行.md](スクリプトの作成と実行.md)｜[条件分岐.md](条件分岐.md)（テスト演算子の詳細）

スクリプトの冒頭の書き方、厳格モード、命名規則、クォート、テスト演算子（-f/-z）などをまとめます。

---

## スクリプトの冒頭

```bash
#!/usr/bin/env bash

# 厳格モード
set -euo pipefail

# スクリプトの説明
# このスクリプトは...

# 定数定義
readonly SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
readonly LOG_FILE="/var/log/script.log"
```

## エラーハンドリング

```bash
#!/bin/bash

# エラーメッセージを標準エラー出力に
error() {
    echo "ERROR: $*" >&2
    exit 1
}

# 使用例
[ -f "$file" ] || error "File not found: $file"
```

## 変数の命名規則

```bash
# 定数: 大文字とアンダースコア
readonly MAX_RETRY=3
readonly CONFIG_FILE="/etc/app.conf"

# 変数: 小文字とアンダースコア
user_name="john"
file_count=0

# 環境変数: 大文字（慣習）
export PATH="/usr/local/bin:$PATH"
```

## クォートの使用

```bash
# 良い例: 変数を必ずクォート
if [ -f "$file" ]; then
    cp "$file" "$destination"
fi

# 悪い例: クォートなし（スペースを含むファイル名で問題）
if [ -f $file ]; then
    cp $file $destination
fi
```

## -f と -z の違い

- **-f** : ファイルの存在チェック

```bash
if [ -f /path/to/file ]; then
  # ファイルが存在し、かつ通常ファイルの場合に実行
fi
```

- **-z** : 文字列が空かチェック

```bash
if [ -z "$VAR" ]; then
  # 変数が空（長さゼロ）の場合に実行
fi
```

### よく使うテスト演算子

| 演算子 | 意味 |
|--------|------|
| `-f` | ファイルが存在する |
| `-d` | ディレクトリが存在する |
| `-e` | パスが存在する（ファイル/ディレクトリ問わず） |
| `-z` | 文字列が空 |
| `-n` | 文字列が空でない |
| `-s` | ファイルが存在し、サイズが0より大きい |

## プライベートnpmレジストリ（Registry）へのアクセス

Registry や Verdaccio などプライベートnpmレジストリにアクセスする場合、`.npmrc` とトークン（環境変数）を組み合わせて使います。

```bash
# 例: 環境変数でトークンを渡し、.npmrc にレジストリURLを記載
# .npmrc（プロジェクトまたはホームの ~/.npmrc）
# registry=https://registry.example.com/
# //registry.example.com/:_authToken=${NPM_TOKEN}
```

CIやスクリプトでは、トークンを環境変数 `NPM_TOKEN` などに設定してから `npm install` を実行します。
