# Scalaの生まれ

Scalaは2003年にスイス連邦工科大学ローザンヌ校（EPFL）のMartin Odersky教授によって開発されたプログラミング言語です。Scalaではオブジェクト指向と関数型プログラミングの両方を行えるところに特徴があります。
また、処理系はJVM上で動作するため1、Java言語のライブラリのほとんどをシームレスに利用することができます。

オブジェクト指向の概念で、関数型の機能を解釈して取り込んでいると言える。

## 変数宣言

Scalaでは、変数を宣言する際に `val` または `var` を使用します。

- **`val` (Immutable - 不変)**: 再代入不可能な変数を宣言します。一度値を代入すると、後から変更することはできません。Javaの `final` 変数に相当します。
- **`var` (Mutable - 可変)**: 再代入可能な変数を宣言します。後から値を変更することができます。

```scala
val x: Int = 3 * 2
```

### 原則として `val` を使用する

Scalaにおいては、**基本的には `val` を使用し、可能な限り不変（Immutable）な設計にすることが強く推奨されます**。

理由は以下の通りです：

1. **コードの予測可能性向上**: 値が変わらないことが保証されるため、ロジックが追いやすくなります。
2. **スレッド安全性**: 複数のスレッドから安全にアクセスでき、バグの混入を防げます。
3. **関数型プログラミングの核**: 状態の変化（副作用）を限定することで、メンテナンス性の高いコードが書けます。

どうしても状態を保持・変更し続けなければならないケース（ループ内での一時的な蓄積など）を除き、まずは `val` で書けないか検討する文化があります。
