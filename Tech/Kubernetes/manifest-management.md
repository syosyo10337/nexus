# マニフェスト管理ツール (Helm vs Kustomize)

Kubernetes クラスタにアプリケーションをデプロイする際、素の YAML ファイル（Raw YAML）をそのまま管理し続けるのは難しくなります。これを解決するために **Helm** や **Kustomize** といった管理ツールが利用されます。

## なぜ管理ツールが必要か？

1. **環境ごとの差異の吸収**: 開発（dev）、検証（stg）、本番（prod）で、レプリカ数や環境変数、リソース割り当てなどを柔軟に変更する必要があります。
2. **再利用性の向上**: 共通の定義（テンプレート）を使い回し、冗長な記述を減らしたい。
3. **管理の複雑化への対応**: 数十、数百の YAML ファイルを効率的に管理・適用したい。

---

## Helm (テンプレート型)

Helm は「Kubernetes のパッケージマネージャー」と呼ばれます。**Chart** という単位でマニフェストをパッケージ化して管理します。

### Helm の特徴

- **テンプレートエンジン**: Go テンプレート形式を用いて、YAML 内に変数や `if`, `range` などのロジックを記述できます。
- **Release 管理**: デプロイごとに履歴が管理され、簡単にロールバック（`helm rollback`）が可能です。
- **ArtifactHub**: 公開されている Chart（Nginx, Prometheus など）を簡単に自分のクラスタに導入できます。

### Helm のメリット

- アプリケーション一式を一つの単位としてデプロイ・管理できる。
- 複雑な条件分岐が可能で、非常に柔軟なマニフェスト作成ができる。
- ライフサイクルの管理（install, upgrade, uninstall）が容易。

### Helm のデメリット

- テンプレート記法が独自で、YAML が読みづらくなることがある（YAML 自体が有効な YAML でなくなる場合がある）。
- 学習コストが Kustomize よりやや高い。

---

## Kustomize (パッチ型 / オーバーレイ型)

Kustomize は「YAML を直接加工する」アプローチを取るツールです。`kubectl` に標準で組み込まれています。

### Kustomize の特徴

- **Base と Overlay**: 共通の `base` 定義に対して、各環境（`overlays/dev`, `overlays/prod`）で差分（パッチ）を当てる形式です。
- **素の YAML**: テンプレートエンジンを使わないため、マニフェストは常に有効な YAML です。

### Kustomize のメリット

- **学習コストが低い**: テンプレート言語を覚える必要がなく、YAML の構造さえ分かれば扱える。
- **kubectl との親和性**: `kubectl apply -k` でそのまま適用できる。
- **可読性が高い**: テンプレート変数が混ざらないため、最終的に出力される YAML が想像しやすい。

### Kustomize のデメリット

- **ロジックを持てない**: 「あるフラグが True のときだけこのリソースを作る」といった複雑な条件分岐ができない。
- **Release の履歴管理がない**: ツール自体にはデプロイ履歴を管理する機能はない（GitOps ツール等と併用するのが一般的）。

---

## Helm と Kustomize の比較

| 特徴 | Helm | Kustomize |
| :--- | :--- | :--- |
| **アプローチ** | テンプレート方式 (変数埋め込み) | オーバーレイ方式 (差分上書き) |
| **主な用途** | 汎用的なパッケージの配布・管理 | 自社アプリの環境ごとの管理 |
| **学習コスト** | 中程度 (Go テンプレート) | 低い (素の YAML) |
| **柔軟性** | 非常に高い (ロジック利用可) | 限定的 (静的な置換・追加) |
| **対応ツール** | 専用バイナリ `helm` | `kubectl` に内蔵 |

## どっちを使うべき？

- **サードパーティ製のツールを導入する場合**: ほとんどが Helm Chart として提供されているため、**Helm** が最適です。
- **自社開発のシンプルなアプリケーション**: 複雑なロジックが不要であれば、学習コストが低く管理がしやすい **Kustomize** がおすすめです。
- **併用**: 最近では、Helm でテンプレートを出力し、その結果を Kustomize で微調整する（Argo CD の機能など）「ハイブリッド構成」も増えています。
