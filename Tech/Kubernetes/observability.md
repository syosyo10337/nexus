# オブザーバビリティ (Observability)

オブザーバビリティとは、システムの外部から出力を観察することで、システムの内部状態をどれだけ理解できるかという能力を指します。Kubernetes のような分散システムでは、単なる「死活監視」を超えた深い洞察が求められます。

## 3 つのプライマリーシグナル (The Three Pillars)

伝統的に「オブザーバビリティの 3 本の柱」と呼ばれている重要なシグナルです。

### 1. ログ (Logs)

- **概要**: システム内で発生した個別のイベントを、タイムスタンプ付きで記録した不変のテキストデータです。
- **特徴**: 「いつ、何が起きたか」という詳細な文脈を追うのに最適です（例：エラーメッセージ、アクセスログ）。
- **課題**: データ量が膨大になりやすく、ストレージコストや検索パフォーマンスが課題になることがあります。

### 2. メトリクス (Metrics)

- **概要**: 一定期間にわたって測定された数値データ（CPU 使用率、リクエスト数、レイテンシなど）です。
- **特徴**: 統計的な傾向の把握やアラートの発報に最適です。ログに比べてデータ量が抑えられ、長期保存に向いています。
- **キーワード**: カウンタ、ゲージ、ヒストグラム。

### 3. トレース (Traces)

- **概要**: 分散システムにおいて、一つのリクエストが複数のマイクロサービスを跨いで処理される経路を可視化したものです。
- **特徴**: サービス間の依存関係や、どこでボトルネックが発生しているかを特定するのに不可欠です。
- **キーワード**: Span, Trace ID, Distributed Tracing.

---

## 2 つの新規シグナル (New Signals)

近年、より深いトラブルシューティングのために重要視されているシグナルです。

### 4. プロファイル (Profiles / Continuous Profiling)

- **概要**: コードレベルでのリソース使用状況（どの関数が CPU を消費しているか、メモリを確保しているかなど）を継続的に収集したデータです。
- **特徴**: 「なぜメトリクスが高いのか」をコードレベルで特定できます。従来のプロファイリングは負荷が高いものでしたが、現在は低負荷な「継続的プロファイリング」が普及しています。

### 5. ダンプ (Dumps)

- **概要**: 特定の時点におけるシステムのメモリ状態やプロセス状態をそのまま書き出したものです（Heap Dump, Thread Dump, Core Dump など）。
- **特徴**: 再現が困難なメモリリークや、デッドロック、クラッシュした際の原因究明に究極の証拠を提供します。ただし、ファイルサイズが非常に大きくなる傾向があります。

## まとめ

| シグナル | 主な目的 | 解決する問い |
| :--- | :--- | :--- |
| **Logs** | 詳細な文脈・履歴 | 何が起きたのか？ |
| **Metrics** | 傾向把握・通知 | 今、異常はあるか？ |
| **Traces** | 依存関係・遅延特定 | どこで遅れているのか？ |
| **Profiles** | コードの効率化 | どの処理がリソースを食っているか？ |
| **Dumps** | 致命的エラーの解析 | メモリの中で何が起きていたのか？ |
